# LocalCut Pro 付费价值产品规划

版本：V1.0
日期：2026-06-08
目标：以现有 MVP 为基准，围绕“本地电商批量切图与素材交付工作台”提升竞争力、可复用性和付费转化。

## 1. 战略判断

LocalCut Pro 当前已经具备基本闭环：导入图片、创建区域、批量生成区域、模板复用、统一尺寸、命名规则、预览和 ZIP 导出。下一阶段不应继续横向堆叠图片编辑能力，而应把产品从“能用的裁剪工具”推到“可依赖的电商素材生产工具”。

核心竞争力应集中在四件事：

- 批量效率：同样一批商品图，用户用更少步骤完成。
- 交付确定性：导出结果、命名、尺寸和预览一致，减少返工。
- 模板复用：同版式图片能一键复用布局、尺寸和命名策略。
- 本地隐私：图片、模板和项目都可本地处理，降低敏感商品素材上传顾虑。

不建议把 AI 抠图、超分、复杂形状和多图层编辑做成主叙事。它们可以作为增强能力存在，但付费理由应来自“批量交付省时间、省返工、可复用”。

## 2. 目标用户与付费动机

### 2.1 主用户

电商运营、店铺美工、内容运营和小团队设计助理。

高频任务：

- 从商品组合图、详情长图、素材拼图中批量切出可上传图片。
- 按店铺或平台要求统一输出 800x800、1080x1080、750x1000 等规格。
- 将同一版式的多张图片重复切片、重命名、打包。
- 在交付前预览所有裁剪结果，减少上传平台后返工。

付费动机：

- 每天或每周重复处理图片，省下的时间能直接感知。
- 本地处理，不想把商品素材上传到第三方工具。
- 模板、命名和批量队列能形成个人或团队流程资产。
- 输出结果稳定，减少错尺寸、错命名、漏导出。

### 2.2 次级用户

素材整理、教学课件制作、游戏素材切片、社媒内容整理用户。

这些用户可以贡献使用量，但第一阶段不应为他们改变核心产品设计。只要批量切图和模板系统足够通用，他们自然可以使用。

## 3. 北极星指标

北极星指标：**每周成功批量导出的有效图片数**。

定义：

- 成功批量导出：用户导出 ZIP，且 ZIP 内至少包含 2 张图片。
- 有效图片数：ZIP 内成功生成的图片数量。

辅助指标：

- 首次导入到首次批量导出的耗时。
- 每批平均导出图片数。
- 模板保存率：完成批量导出后保存模板的比例。
- 模板复用率：套用模板后完成批量导出的比例。
- 命名规则使用率：默认规则被修改或变量被插入的比例。
- 批量预览使用率：导出前打开批量预览的比例。
- 返工代理指标：同一项目短时间内重复导出次数。

## 4. 产品分层

### 4.1 免费层

目标：让用户完整体验核心价值，但在高频批量和复用上遇到自然升级点。

建议包含：

- 单张图片导入。
- 手动创建区域。
- 少量区域批量导出，例如每次最多 5 个区域。
- 基础统一尺寸。
- 默认命名规则。
- 基础模板保存，例如最多 3 个模板。
- 智能抠图可试用低频次数或仅保留基础模型。

### 4.2 Pro 层

目标：围绕“批量、省时、可复用、交付稳定”收费。

建议包含：

- 不限区域数量批量导出。
- 多图片任务队列。
- 模板导入/导出、分类、搜索。
- 项目保存/打开。
- 高级命名规则和导出清单。
- 批量预览、错误检查和导出前体检。
- 可维护平台预设。
- AI 抠图、超分、锐化作为增强导出能力。

### 4.3 团队/专业版（后置）

仅在 Pro 层验证后考虑。

可能包含：

- 团队模板包。
- 品牌/店铺规格配置。
- 批量处理历史。
- 本地文件夹监听。
- 内网部署或离线授权。

第一阶段不做账号协作和云端素材库，避免把产品复杂度拉高。

| P1 复用资产 | 让模板成为生产资产 | 提升留存 | 模板导入导出、模板携带导出配置、模板管理 |
| P2 项目连续性 | 让用户能分多次完成工作 | 提升专业感 | 项目保存/打开、最近项目、恢复工作区 |

## 6. P0：可靠性交付

目标：用户看到什么，导出就是什么；用户设置什么，单张和批量就一致执行。

### 6.1 图层预览与导出顺序回归保护

当前判断：现有实现已经基本一致。画布绘制和导出合成都按图层数组的反向顺序绘制，使图层列表顶部成为视觉最上层；活动图层只额外绘制独立外框和控制点，不改变实际图片层级。

风险：后续增加项目保存、多图片任务队列、批量预览重构或导出体检时，可能无意破坏这一语义。因此这项不应作为待开发大功能，而应作为回归测试和架构约束。

功能点：

- 文档中明确图层语义：图层列表顶部为最上层。
- 画布、批量预览、放大预览、单区域导出、批量导出继续共用同一合成顺序。
- 活动图层只绘制独立外框和控制点，不改变实际渲染层级。
- 为重叠图层增加人工验收用例或自动回归用例。

验收：

- 两个重叠图层在画布、单预览、批预览、单导出、批导出中遮挡关系完全一致。
- 切换活动图层不会改变遮挡关系。

优先级：P0 回归保护。

### 6.2 上传入口自动适配画布

问题：拖拽导入会适配画布，按钮上传没有显式适配，影响首次体验。

功能点：

- 左侧上传、菜单导入、拖拽导入统一走同一个添加图片入口。
- 首张图片导入后自动 fitToCanvas。
- 多张图片导入时，视图优先适配当前活动图层或全部图层外接矩形。

验收：

- 首次点击上传后，图片完整出现在画布可视区域。
- 菜单导入和拖拽导入表现一致。

优先级：P0 必做。

### 6.3 单区域导出复用命名规则

当前状态：已提供“单张也使用命名规则”开关，默认可让单区域导出复用命名规则。

剩余问题：UI 重排和导出流程解耦后，仍需要持续回归单区域导出、批量导出、预览文件名和体检结果的一致性。

功能点：

- 单区域导出默认也使用同一套命名变量。
- 单区域导出上下文中 `{index}` 默认为 1 或当前区域在列表中的序号。
- 文件名清洗、去重逻辑复用批量导出。
- UI 上提供“单张也使用命名规则”的开关，默认开启。

验收：

- 同一区域单独导出和批量导出时，在同一命名规则下文件名可预测。
- 非法字符被清洗。

优先级：P0 回归保护。

### 6.4 导出前体检

目标：减少用户导出后才发现尺寸、命名、空区域或低清晰度问题。

当前状态：基础体检能力已经具备，下一步重点是摘要同步、问题定位和批量预览联动。

功能点：

- 检查区域数量、勾选数量、输出尺寸、文件名重名、未知变量。
- 检查区域是否超出活动图层或没有覆盖任何可见图层。
- 检查源像素不足时提示 DPR 已被限制或建议启用超分。
- 批量导出按钮旁显示“可导出 N 项 / 有 M 个警告”。
- 体检面板可点击定位问题区域。

验收：

- 未知命名变量阻止导出或明确要求确认。
- 空区域、重名、超出图层等问题能在导出前看到。

优先级：P0 完善。

## 7. P1：模板资产化

目标：模板从“本地小工具”升级为用户愿意付费保存和迁移的生产资产。

### 7.1 模板导入/导出

当前状态：已支持模板 JSON 导入/导出，以及名称冲突处理。

功能点：

- 导出单个模板为 `.localcut-template.json`。
- 导出多个模板为 `.localcut-templates.json`。
- 导入时检查版本、模板名称冲突和字段完整性。
- 冲突处理：覆盖、重命名、跳过。

验收：

- 一台机器保存模板，另一台机器导入后可套用。
- 旧版本模板能给出兼容提示。

优先级：P1 体验完善与回归。

### 7.2 模板携带导出配置

当前状态：已支持保存尺寸、适配方式、背景色、命名规则、格式和质量，并可选择“只套区域”或“区域 + 导出设置”。

剩余问题：运营真正复用的不只是区域布局，还包括配置套用后的状态摘要、导出体检和底部交付栏一致性。

功能点：

- 保存模板时可选择保存：区域布局、批量输出尺寸、适配方式、背景色、命名规则、格式质量。
- 套用模板时提供“只套区域”或“区域 + 导出设置”。
- 模板列表显示关键摘要，例如 `12区 / 800x800 / cover / PNG`。

验收：

- 用户保存一套商品宫格模板后，换图套用即可直接导出同规格 ZIP。
- 不想覆盖当前导出设置时可只套区域。

优先级：P1 回归验收与体验优化。

### 7.3 模板管理

功能点：

- 模板分类：通用、电商主图、详情长图、店铺装修、自定义。
- 搜索模板名称。
- 最近使用排序。
- 收藏模板。
- 复制模板。
- 删除前确认。

验收：

- 模板数量超过 20 个时仍能快速找到。
- 最近使用的模板出现在靠前位置。

优先级：P1 应做。

## 8. P2：项目保存与恢复

目标：让 LocalCut Pro 从一次性网页工具变成可持续工作的桌面生产工具。

### 8.1 项目文件 `.localcut`

功能点：

- 保存当前工作区为项目文件。
- 项目包含：图层元数据、图片数据或图片引用、区域、文字、参考线、导出设置、选中状态、视图状态。
- 支持打开项目文件恢复工作区。
- Electron 版本优先支持本地文件保存；浏览器版本可下载/上传项目 JSON。

建议格式：

```json
{
  "version": 1,
  "app": "LocalCut Pro",
  "createdAt": 0,
  "updatedAt": 0,
  "layers": [],
  "regions": [],
  "texts": [],
  "guides": {},
  "exportSettings": {},
  "view": {}
}
```

验收：

- 保存后关闭应用，再打开项目，区域、图层、导出设置保持一致。
- 打开不兼容版本时给出明确提示。

优先级：P2 必做。

### 8.2 最近项目与自动恢复

功能点：

- 最近打开项目列表。
- 未保存修改提示。
- 异常关闭后恢复最近一次工作区草稿。
- 草稿只保存在本地用户数据目录。

验收：

- 关闭前有未保存改动时提示保存。
- 意外关闭后可以恢复上次工作。

优先级：P2 应做。

## 9. P3：多图片任务队列

目标：把“模板复用”进一步变成“多张同版式图一键交付”，这是最强付费价值点。

### 9.1 多图片批量导入为任务

功能点：

- 用户选择多张图片进入任务队列。
- 每张图片是一个 job，而不是一个叠在画布里的图层。
- 任务字段：图片名、状态、套用模板、区域数、警告数、导出结果。
- 支持删除、排序、跳过失败任务。

验收：

- 导入 20 张图不会变成 20 个叠加图层。
- 每张图能独立预览和导出。

优先级：P3 必做。

### 9.2 队列套模板

功能点：

- 选择一个模板，一键套用到全部任务。
- 支持对单个任务微调区域。
- 支持“从当前任务同步区域到其他任务”。
- 支持任务间快速切换，上一个/下一个。

验收：

- 同版式 20 张图，可在 1 次选择模板后生成全部区域。
- 单张微调不影响其他任务，除非用户显式同步。

优先级：P3 必做。

### 9.3 队列批量导出

功能点：

- 每张图片导出为独立文件夹。
- 支持扁平 ZIP 或按图片名分文件夹 ZIP。
- 文件命名变量新增 `{jobIndex}`、`{jobName}`。
- 导出完成后显示成功、失败和警告统计。

验收：

- 20 张图，每张 8 个区域，最终 ZIP 内结构清晰且不重名。
- 单个任务失败不影响其他任务导出。

优先级：P3 必做。

## 10. P4：专业交付能力

目标：让产品更像“交付工具”，而不是只会生成图片。

### 10.1 平台规格包

问题：平台规则会变，不宜写死为官方真理。

功能点：

- 内置通用规格包。
- 支持用户自定义平台、尺寸、命名建议和备注。
- 支持导入/导出规格包。
- 选择规格后填充尺寸，但不强制覆盖命名规则。

验收：

- 用户可以创建自己的“淘宝主图”“店铺 A 详情图”“小红书封面”等规格。
- 规格变化时用户可以自行修改。

优先级：P4 应做。

### 10.2 导出报告

功能点：

- 导出 ZIP 时附带 `export-report.csv` 或 `export-report.json`。
- 记录原图、区域名、输出文件名、宽高、格式、警告。
- 可关闭。

验收：

- 用户能核对本次交付了哪些文件。
- 出现警告时报告中能追踪到具体区域。

优先级：P4 可做。

### 10.3 参考线与吸附增强

功能点：

- 参考线管理面板：新增、删除、清空、重命名。
- 区域移动和缩放时吸附参考线、图层边界、其他区域边界。
- 支持等分参考线，例如三等分、四等分。

验收：

- 规则版式切图时无需手动输入大量坐标。
- 吸附可开关，避免干扰精细调整。

优先级：P4 应做。

## 11. P5：AI 增强但不喧宾夺主

目标：AI 服务于批量交付，而不是把产品变成泛 AI 修图工具。

### 11.1 抠图结果进入模板与批量流程

功能点：

- 抠图结果可作为当前任务图层。
- 支持批量队列中对每张图执行基础抠图，但需要明确耗时和失败处理。
- 抠图结果可以参与统一尺寸和背景色适配。

验收：

- 用户抠图后可以继续使用模板、命名和 ZIP 导出。
- 抠图失败不会破坏原图任务。

优先级：P5 应做。

### 11.2 导出增强预设

功能点：

- 提供“原样导出”“轻微锐化”“电商清晰”“低清晰源图增强”等预设。
- 将超分、锐化和 DPR 组合成用户可理解的选项。
- 高级参数仍可展开调整。

验收：

- 非专业用户无需理解 DPR、USM、超分模型，也能选对策略。

优先级：P5 可做。

## 13. 版本路线图

### V1.3：可靠性交付版

目标：现有 MVP 可信、稳定、可交付。

范围：

- 上传入口自动适配。
- 单区域导出命名一致性回归。
- 批量预览导出触发解耦。
- 基础导出前体检摘要和定位优化。
- 图层预览与导出顺序增加回归验收。
- 更新产品文案和快捷键说明。

可付费意义：

- 减少返工，建立信任。

### V1.4：模板资产版

目标：模板成为可迁移、可复用的生产资产。

范围：

- 模板导入/导出体验完善。
- 模板保存导出配置回归验收。
- 模板分类、搜索、收藏、复制和最近使用体验优化。
- 平台规格自定义。

可付费意义：

- 模板数量、模板包、导出配置复用可作为 Pro 权益。

### V1.5：项目连续性版

目标：用户可以分多次完成复杂工作。

范围：

- 项目保存/打开。
- 最近项目。
- 未保存改动提示。
- 自动恢复草稿。

可付费意义：

- 项目文件和恢复能力明显提升专业工具感。

### V1.6：多图队列版

目标：处理同版式多图片，这是付费转化的关键版本。

范围：

- 多图片任务队列。
- 队列套模板。
- 每任务预览和微调。
- 按图片分文件夹导出 ZIP。
- 导出报告。

可付费意义：

- 明确区分免费裁剪工具和 Pro 批量生产工具。

## 14. 不做清单

短期不做：

- 云端同步。
- 多人协作。
- 模板市场。
- 全功能图片编辑器。
- 视频处理。
- 移动端触控优先。
- 复杂平台官方规则实时更新。
- AI 生成商品图。

这些能力不是没有价值，而是会稀释当前最清晰的付费路径。

## 15. 商业化建议

### 15.1 首个可收费版本

建议从 V1.4 或 V1.5 开始收费验证，而不是等到所有功能完成。

最小 Pro 包：

- 不限批量导出数量。
- 模板导入/导出和模板包管理体验。
- 模板保存导出配置和跨图复用能力。
- 项目保存/打开。
- 高级命名规则。
- 导出前体检。

### 15.2 免费限制建议

限制应该贴近价值，而不是粗暴限制基础使用。

可限制：

- 每次批量导出区域数。
- 模板数量。
- 项目保存数量或是否可保存项目。
- 多图片任务队列。
- 模板包导入/导出。

不建议限制：

- 单张手动裁剪。
- 基础预览。
- 基础导出。
- 本地隐私能力。

## 16. 风险与验证

主要风险：

- 用户是否真的有高频批量切图需求。
- 多图片队列复杂度是否高于收益。
- 模板保存导出配置是否会让 UI 变复杂。
- 本地项目文件若包含图片，会导致文件体积较大。

验证方式：

- 找 5-10 位电商运营或店铺美工试用，观察一批 20 张图的完整处理过程。
- 记录从导入到 ZIP 导出的耗时和返工点。
- 让用户说出愿意付费的具体功能，而不是问“你会不会付费”。
- 对比无模板、单模板、多图片队列三种流程的时间节省。

## 17. 一句话方向

LocalCut Pro 应继续成为“本地、可靠、可复用的电商批量切图交付工具”，付费价值来自批量效率和模板资产，而不是来自功能数量。
