# LocalCut Pro 产品整改工作清单

版本：V1.0
日期：2026-06-09
目标：基于当前代码实现和产品目标，梳理后续真正需要整改、打磨和规划的工作项，避免重复推进已经完成的 UI 迁移。

## 0. 产品目标

LocalCut Pro 的目标不是成为轻量 Photoshop、在线 Canva 或泛 AI 修图工具，而是成为一个能收费的本地批量切图与图片交付工具。

一句话目标：

> 让熟悉 Figma、Photoshop 或类似设计工具的用户，用更少步骤、更少学习成本，把一批图片稳定切成可交付文件。

用户愿意付费的理由应该来自：

- 批量效率：少重复框选、少重复设置、少重复导出。
- 交付稳定：预览、体检、命名、尺寸和 ZIP 结果一致。
- 模板资产：切图布局和导出配置可保存、迁移、复用。
- 本地隐私：图片、模板、项目和导出尽量在本地完成。
- 迁移顺滑：Figma/PS 用户能自然理解画布、对象、属性、模板、导出。

## 1. 当前实现基线

以下能力已经具备，后续整改必须在此基础上打磨，不要重新设计成待迁移事项。

### 1.1 已完成的 UI 骨架

- `LeftSidebar.vue` 已经是窄工具栏，包含导入、AI 抠图、选择、画笔、橡皮、魔棒、图形、文字。
- `RightSidebar.vue` 已经使用 `属性 / 批量 / 模板 / 导出` 四个 Tab。
- 右侧底部已经有可拖拽、可收起的卡片列表，裁剪框和文字框分组展示。
- `DeliveryBar.vue` 已经固定在底部，承载导出数量、导出摘要、体检状态、批量预览、导出 ZIP。
- `PreviewModal.vue` 已经通过 `emit('export')` 调用统一导出动作，不再依赖 DOM 查询按钮。

### 1.2 已完成的生产能力

- 导入图片后会调用 `fitToCanvas()`，拖拽导入、按钮上传、Electron 打开文件都已经接近一致。
- 批量生成包括快捷尺寸、网格生成、等距复制、长图切片、参考线切片。
- 快捷尺寸已经在 `BatchCutPanel.vue`，并支持自定义尺寸导入/导出。
- 模板已经支持保存区域、可选保存导出配置、分类、搜索、收藏、复制、导入/导出、同名冲突处理。
- 导出已经包含格式、质量、DPR、统一尺寸、适配方式、背景色、命名规则、体检、超分、锐化。
- 批量导出动作已经抽到 `useBatchExportWorkflow()`，底部交付栏和批量预览共用同一流程。

### 1.3 当前不要重复做的事

- 不要再把左侧瘦身、右侧 Tabs、底部交付栏当作待办。
- 不要为了“组件更干净”优先拆 `ObjectPanel.vue`、`PropertiesPanel.vue`，除非维护成本已经明显影响开发。
- 不要新增第二套批量导出入口或绕过 `useBatchExportWorkflow()`。
- 不要把 AI 抠图、超分、复杂形状推到主叙事前面。

## 2. 优先级规则

P0：直接影响用户是否相信结果、是否能完成交付、是否会返工。

P1：直接影响付费价值、复用效率、专业工具感。

P2：影响中期留存和 Pro 版本差异化，但不阻塞当前单图/多区域闭环。

P3：增强项或商业化后置项，必须等核心交付稳定后再做。

## 3. P0：交付可信度整改

### P0-1 批量预览信息不足

现状：批量预览已经能展示缩略图，并能通过事件触发批量导出，但预览卡片主要显示区域名，尚未充分展示最终交付信息。

问题：用户导出前看不到完整文件名、输出尺寸、格式、体检警告时，会不确定 ZIP 里到底是什么，影响付费信任。

整改规划：

- 批量预览卡片显示最终文件名，而不只是区域名。
- 显示输出尺寸和格式，例如 `800x800 PNG`。
- 显示警告标记，例如源像素不足、越界、重名风险。
- 批量预览使用和 `DeliveryBar` 一致的目标集合：有勾选导出勾选项，未勾选导出全部。
- 批量预览中的确认导出继续调用统一导出动作。

验收标准：

- 用户在批量预览中能看懂每一项将导出为什么文件。
- 批量预览数量、底部交付栏数量和 ZIP 内文件数量一致。
- 有体检警告时，预览中能看到对应项的警告提示。

### P0-2 导出体检缺少问题定位闭环

现状：已经有导出体检能力，`DeliveryBar` 也能显示可导出、警告、错误。

问题：如果用户只知道“有 N 个警告”但不能快速定位到区域或设置项，体检价值会下降。

整改规划：

- 体检详情中的区域问题支持点击后选中并定位区域。
- 命名、格式、尺寸类问题引导用户进入导出 Tab 的对应区域。
- 底部体检状态点击后优先打开体检详情，而不是只展示摘要。
- 阻断错误和非阻断警告视觉区分清晰。

验收标准：

- 点击体检问题能定位到具体区域或具体设置。
- 阻断错误会禁用导出；警告允许导出但用户可见。
- 用户不用猜测问题来源。

### P0-3 导出完成状态不足

现状：导出中已有遮罩和状态文案，导出完成后缺少明确的成功反馈。

问题：桌面生产工具需要让用户确认“刚才这批文件已经交付成功”，否则容易重复点击或不信任结果。

整改规划：

- `DeliveryBar` 导出完成后显示短暂成功状态，例如 `已导出 N 项`。
- 浏览器版本显示 `重新导出`。
- Electron 版本优先增加 `打开位置`。
- 导出失败时显示失败原因，而不是只 `console.error`。

验收标准：

- 成功导出后有明确 UI 状态。
- 导出失败时用户知道是否需要重试、改设置或减少导出范围。
- 导出中无法重复点击。

### P0-4 单区域导出与批量导出一致性回归

现状：单区域导出已经支持使用命名规则，批量导出走统一规则。

问题：单区导出、批量导出、批量预览、体检若使用不同上下文，会造成文件名或尺寸不一致。

整改规划：

- 单区域导出继续使用导出 Tab 的格式、命名、增强配置。
- 单区域体检和批量体检共享同一套规则，只在数量和重名语义上区分。
- 验证 `{index}` 在单区域导出中的结果可预测。

验收标准：

- 同一区域单独导出和批量导出时，命名、尺寸、格式逻辑一致。
- 体检结果不会在单区和批量之间互相矛盾。

### P0-5 首次使用路径需要更短

现状：主界面功能已经完整，但新用户首次进入时仍可能不知道第一步应该做什么。

问题：目标用户虽然熟悉 Figma/PS，但不代表愿意阅读工具说明。第一步必须非常清楚。

整改规划：

- 空画布提供可点击 `导入图片` 主按钮，复用现有上传入口。
- 空状态只保留一句主提示和支持格式，不展示长说明。
- 未导入图片时，右侧面板显示简短空状态，不堆禁用表单。

验收标准：

- 新用户 10 秒内能完成导入。
- 无图片状态下界面不显得“坏掉”或复杂。

## 4. P1：付费价值打磨

### P1-1 模板资产库表达还可以更清楚

现状：模板功能已经比较完整，支持区域和导出配置保存、导入导出、分类、搜索、收藏、复制。

问题：用户需要明确知道模板不是装饰预设，而是生产资产。否则它的付费价值不容易被感知。

整改规划：

- 模板项摘要固定展示 `区域数 / 输出尺寸 / 适配方式 / 格式`。
- 保存模板时明确“区域”为必选，“尺寸、适配、背景、命名、格式”为可选。
- 套用时继续保留 `只套区域` 与 `区域+配置`，并在套用后同步更新底部交付栏摘要。
- 导入模板后显示导入结果、冲突处理结果和可套用数量。

验收标准：

- 用户能理解模板可以跨图复用布局和导出配置。
- 换图套模板后可以直接进入预览和导出。
- 模板文件在另一台机器导入后可用。

### P1-2 快捷尺寸与导出尺寸需要防混淆

现状：快捷尺寸已放在批量 Tab，导出尺寸在导出 Tab，位置已经正确。

问题：用户可能混淆“创建裁剪框尺寸”和“最终输出尺寸”。这会导致导出结果和预期不一致。

整改规划：

- 批量 Tab 中把快捷尺寸文案稳定为“创建裁剪框”语义。
- 导出 Tab 中把输出尺寸文案稳定为“最终文件尺寸”语义。
- 在 `DeliveryBar` 摘要中只展示最终导出尺寸，不展示快捷尺寸。
- 自定义快捷尺寸导入/导出保留，但不要叫“平台规格”。

验收标准：

- 用户能区分裁剪框尺寸和导出文件尺寸。
- 平台预设只影响导出尺寸，不影响当前裁剪框。

### P1-3 顶部设置要继续收敛

现状：顶部设置包含裁剪约束、图层名称展示、导入排列方向和间距。

问题：顶部如果继续增加局部配置，会破坏“画布工具在左、对象属性在右、交付在底”的心智。

整改规划：

- 顶部只放品牌、设置、快捷键，项目文件能力完成后再加入项目级状态。
- 不把导出体检、批量参数、模板入口放到顶部。
- 设置项保持系统级，不承载对象级配置。

验收标准：

- 顶部始终轻量。
- 高频生产动作仍在右侧和底部完成。

### P1-4 视觉语言继续统一

现状：整体已经转向石墨灰 + 绿色主操作，但部分区域仍可能有字符图标、蓝色 hover 或卡片层级不一致。

问题：一个能卖钱的工具需要稳定、专业、低噪音的视觉语言。

整改规划：

- 继续减少字符图标和符号按钮，优先使用统一线性图标。
- 主操作使用绿色，画布空间辅助使用蓝色，警告琥珀色，危险红色。
- 列表行、按钮、输入框高度保持稳定。
- 模板资产库避免卡片套卡片。

验收标准：

- 界面第一眼像生产工具，而不是演示页或 AI 玩具。
- 长名称、长文件名不会挤压按钮或导致布局跳动。

## 5. P2：专业工作流规划

### P2-1 `.localcut` 项目保存/打开

理由：项目保存是从“网页工具”变成“专业生产工具”的关键一步，也是 Pro 付费点。

规划：

- 保存图层元数据、图片数据或图片引用、裁剪区域、文字、参考线、导出配置、视图状态。
- Electron 版本优先支持本地文件保存和打开。
- 浏览器版本可通过下载/上传项目 JSON 实现。
- 保存前明确文件体积风险。

验收标准：

- 关闭后重新打开项目，图层、区域、文字、参考线、导出配置保持一致。
- 不兼容版本给出明确提示。

### P2-2 多图片任务队列

理由：多图同版式处理是最强付费差异点，但不能把多张图继续堆成多图层。

规划：

- 多张图片导入为任务队列，每张图片是一个 job。
- 队列任务支持套模板、单张微调、跳过、删除、查看警告。
- 新增命名变量 `{jobName}`、`{jobIndex}`。
- 支持按图片名分文件夹导出 ZIP。

验收标准：

- 导入 20 张图不会变成 20 个叠加图层。
- 同一模板可以一键套到全部任务。
- 单任务失败不影响其他任务导出。

### P2-3 导出报告

理由：导出报告能让产品更像交付工具，适合电商团队核对结果。

规划：

- ZIP 内可选附带 `export-report.csv` 或 `export-report.json`。
- 记录原图名、区域名、输出文件名、尺寸、格式、警告。
- 体检警告能在报告中追踪。

验收标准：

- 用户能核对本次导出了哪些文件。
- 报告内容和 ZIP 实际文件一致。

### P2-4 平台规格包自定义

理由：平台规则会变化，写死官方规则会带来维护和信任风险。

规划：

- 内置常用规格继续保留为“常用尺寸”。
- 支持用户自定义平台、尺寸、备注。
- 支持规格包导入/导出。
- 选择规格只填充导出尺寸，不覆盖命名规则。

验收标准：

- 用户能创建自己的店铺规格。
- 规格变化时用户可自行维护。

## 6. P3：后置增强

### P3-1 AI 能力包装为导出增强

理由：AI 抠图、超分、锐化有价值，但不应抢占主产品定位。

规划：

- 把超分、锐化、DPR 组合成用户可理解的导出增强预设。
- 高级参数继续可展开。
- 批量 AI 处理必须先明确耗时、失败处理和取消机制。

验收标准：

- 用户无需理解 DPR、USM、模型细节，也能选对增强策略。
- AI 失败不破坏原图任务。

### P3-2 免费/Pro 分层

理由：需要把升级点贴近真实价值，而不是粗暴限制基础使用。

规划：

- 免费层保留单张手动裁剪、基础预览、基础导出、本地隐私。
- Pro 限制优先围绕批量区域数、模板数量、模板导入/导出、项目保存、多图队列。
- 触发限制时解释价值，例如“升级后可一次导出全部 N 项”。

验收标准：

- 免费用户能完整体验闭环。
- 高频用户自然遇到升级理由。
- 升级提示不打断编辑状态。

## 7. 里程碑建议

### M1：当前单图/多区域闭环可信

范围：

- 批量预览补齐最终文件名、尺寸、警告。
- 体检问题可定位。
- 导出完成/失败状态完善。
- 单区导出和批量导出一致性回归。

目标：让用户敢把当前版本用于真实交付。

### M2：模板资产与付费感知

范围：

- 模板资产库表达打磨。
- 快捷尺寸和导出尺寸防混淆。
- 顶部设置收敛。
- 视觉语言统一。

目标：让用户理解模板、命名和导出配置是值得付费的资产。

### M3：项目连续性

范围：

- `.localcut` 项目保存/打开。
- 最近项目。
- 未保存提示。
- 自动恢复草稿。

目标：让 LocalCut Pro 从一次性工具变成桌面生产工具。

### M4：多图队列 Pro 主线

范围：

- 多图片任务队列。
- 队列套模板。
- 每任务预览和微调。
- 按图片分文件夹导出。
- 导出报告。

目标：形成最强付费差异点。

## 8. 不做清单

短期不做：

- 全功能图片编辑器。
- 云端素材库。
- 多人协作。
- 模板市场。
- 移动端触控优先。
- AI 生成商品图。
- 平台官方规则实时更新。

原因：这些方向会稀释当前最清晰的价值主线：本地、批量、模板、交付稳定。

## 9. 成功指标

北极星指标：

- 每周成功批量导出的有效图片数。

辅助指标：

- 首次导入到首次 ZIP 导出的耗时。
- 批量预览使用率。
- 导出体检问题修复率。
- 模板保存率。
- 模板复用后导出率。
- 自定义快捷尺寸创建率。
- 同一项目重复打开率。
- 同一批任务重复导出次数。
