# LocalCut Pro 产品说明书与项目审核报告

版本：V1.2  
日期：2026-06-05  
依据：基于当前工作区代码实现、文档与 `npm run build` 验证

## 1. 产品概述

LocalCut Pro 是一款运行在浏览器端的本地图片素材切割工作台，面向电商运营、内容制作和素材整理场景。用户可以导入一张或多张图片作为图层，在画布中创建、编辑和批量生成裁剪区域，再按统一尺寸、文件命名规则和导出格式批量打包为 ZIP。

产品核心价值是：在不上传图片、不依赖后端的前提下，完成图片局部选区、批量区域生成、轻量标注、模板复用、统一尺寸输出和自动命名导出。

## 2. 产品定位

当前产品定位为“本地电商批量切图与素材裁剪工具”，介于截图裁剪工具、批处理工具和轻量图片编辑器之间。

适合解决：

- 商品主图、详情图、宫格图、素材集的批量切图。
- 多个裁剪区域的快速生成、复用、预览和导出。
- 对局部区域添加文字、画笔和橡皮修改后导出。
- 使用网格、等距复制、长图切片和参考线切片减少重复操作。
- 按平台常用尺寸、统一尺寸策略和文件命名模板交付素材。

暂不承诺为：

- Photoshop、Photopea、Canva 类完整图像编辑器。
- 云端项目管理、多人协作或素材资产库。
- 多图片独立任务队列系统。
- 专业抠图、复杂图像分割或 AI 视觉处理工具。
- 移动端触控优先的图片编辑产品。

## 3. 目标用户

- 电商运营人员：从商品图、详情长图、组合图中批量导出可上传素材。
- 设计与内容制作人员：对图片局部区域进行快速圈选、标注、预览和导出。
- 游戏、教学与素材整理人员：从大图、截图或素材表中提取多个规则或不规则区域。
- 本地隐私敏感用户：希望图片只在浏览器端处理，不经过服务器。

## 4. 核心使用流程

1. 用户上传或拖拽导入图片。
2. 系统将图片作为图层加入画布，新图层自动成为活动图层。
3. 用户选择、移动、缩放、重命名、排序、隐藏或删除图层。
4. 用户用形状、多边形、魔棒或批量生成工具创建裁剪区域。
5. 用户移动、缩放、重命名、复制、删除、多选或变换裁剪区域。
6. 用户可添加文字，或使用画笔/橡皮修改活动图层。
7. 用户可预览单个区域或批量区域。
8. 用户设置导出格式、质量、DPR、批量统一尺寸、适配方式和文件命名规则。
9. 用户导出当前区域，或将全部/勾选区域打包为 ZIP。

## 6. 功能说明

### 6.1 图片与图层

用户可以多次上传或拖拽导入图片。每张图片会创建一个图层，包含名称、原始图片、可编辑工作画布、位置、缩放比例和可见性。

图层支持：

- 点击右侧图层列表切换活动图层。
- 双击图层名称进行重命名。
- 隐藏或显示图层。
- 上移或下移图层顺序。
- 删除图层。
- 在画布中拖拽移动图层。
- 通过活动图层控制点缩放图层。
- 在非选择工具下 Ctrl/Command + 拖拽图层，可快速移动图层。

当前渲染会将活动图层置于画布交互优先位置，便于编辑；导出时按 `layers` 顺序合成可见图层。多图层遮挡场景下，这一点可能导致画布预览和导出结果不完全一致。

### 6.2 裁剪区域

支持创建以下区域：

- 矩形。
- 圆形。
- 三角形。
- 菱形。
- 星形。
- 心形。
- 自定义多边形。
- 魔棒生成的自定义多边形。

区域支持：

- 画布点击选中。
- 右侧列表选中。
- Shift + 点击多选或取消多选。
- Shift + 拖拽框选多个区域。
- 拖拽移动，控制点缩放。
- 右侧输入精确编辑 x/y/宽/高。
- 重命名。
- 删除和清空。
- 复制和粘贴。
- 方向键 1px 微移。
- Shift + 方向键 10px 微移。
- 多选后批量移动、批量删除和批量变换。

自定义多边形支持：

- 连续点击添加顶点。
- 点击起点、双击或 Enter 完成。
- Escape 取消。
- 拖拽顶点调整。
- 双击边线新增顶点。
- 右键顶点删除。

### 6.3 区域变换

当前支持对单个区域或多选区域执行：

- 水平翻转。
- 垂直翻转。
- 左转 90 度。
- 右转 90 度。
- 自定义角度旋转，范围 -360 到 360 度。

变换后区域会转换为自定义多边形，以保留旋转和翻转后的轮廓。

### 6.4 魔棒智能选区

魔棒基于活动图层的工作画布执行颜色连通区域识别：

- 以点击像素为种子点。
- 点击位置会从画布世界坐标转换为活动图层本地坐标，并考虑图层偏移和缩放。
- 根据容差计算颜色距离。
- 使用连通填充生成 mask。
- 追踪区域轮廓并简化轮廓点。
- 将轮廓点转换回画布世界坐标，生成可编辑的自定义多边形区域。

当前魔棒适合背景清晰、颜色边界明显的素材。复杂纹理、渐变、抗锯齿边缘或半透明边界仍可能需要手工修点。

### 6.5 画笔与橡皮

画笔和橡皮作用于当前活动图层的工作画布。

画笔支持：

- 大小：1-100px。
- 颜色：任意颜色。

橡皮支持：

- 大小：1-150px。

画笔和橡皮会处理活动图层的偏移和缩放，编辑结果参与导出。开启“原图”后，画布预览和导出优先使用原始图像，而不是工作画布。

### 6.6 文字标注

文字标注是全局画布对象，不绑定具体图层。文字支持：

- 点击画布创建。
- 选中、移动、缩放文字框。
- 编辑内容。
- 修改字号。
- 设置常规/粗体。
- 修改颜色。

导出区域时，若文字框与区域重叠，文字会被裁剪到区域形状内。

### 6.7 批量切图

批量切图能力面向电商运营场景，目标是减少重复圈选。

网格生成：

- 设置行数、列数。
- 设置起始 X/Y。
- 设置单元宽高。
- 设置横向和纵向间距。
- 设置命名前缀。
- 可选择替换现有区域或追加生成。

等距复制：

- 基于当前选中区域生成多个副本。
- 支持横向、纵向和自定义偏移。
- 多边形区域复制时会同步平移顶点。

长图切片：

- 基于活动图层显示矩形生成纵向切片。
- 支持开始 Y、结束 Y、切片高度、重叠值和命名前缀。
- 最后一片可小于指定高度。

参考线切片：

- 用户通过 Ctrl/Command + 点击标尺添加横向或纵向参考线。
- 根据活动图层边界和参考线交叉区域生成矩形裁剪框。

### 6.8 裁剪模板

模板用于复用一组裁剪区域布局。

当前支持：

- 将当前区域保存为本地模板。
- 模板保存在浏览器 `localStorage`。
- 选择模板并套用到当前活动图层。
- 重命名模板。
- 删除模板。

模板只保存区域布局，不保存图片、图层、文字、导出格式或命名规则。当前没有模板导入/导出、云端同步或模板分类。

### 6.9 预览

单区域预览：

- 对当前选中区域生成预览。
- 支持放大查看。
- 支持滚轮缩放。
- 支持 Space + 拖拽平移。

批量预览：

- 若勾选了区域，则预览勾选区域。
- 若未勾选区域，则预览全部区域。
- 缩略图可点击进入放大预览。
- 预览弹窗内可触发批量导出。

### 6.10 导出

支持格式：

- PNG。
- JPEG。
- WebP。

JPEG 和 WebP 支持质量设置，PNG 不使用质量参数。导出支持 DPR，当前范围为 1x 到 4x。

单区域导出：

- 导出当前选中区域。
- 可使用单区域输出宽高。
- 文件名使用区域名称。

批量导出：

- 若未勾选区域，则导出全部区域。
- 若已勾选区域，则导出勾选区域。
- 输出 ZIP，默认名称为 `output.zip`。
- 支持批量统一输出尺寸。
- 支持适配方式：裁切填满、完整包含、拉伸、原始。
- `contain` 模式支持背景色。
- 支持平台尺寸预设。
- 支持文件命名规则和文件名去重。

当前导出逻辑会合成所有可见图层，再按区域形状裁剪，并绘制区域内文字。

### 6.11 文件命名规则

批量导出支持命名模板：

默认规则：

```text
{imageName}_{regionName}_{index:3}
```

支持变量：

- `{imageName}`：活动图层名称。
- `{regionName}`：区域名称。
- `{index}`：序号。
- `{index:2}`、`{index:3}`：补零序号。
- `{width}`：输出宽度。
- `{height}`：输出高度。
- `{format}`：输出格式。
- `{date}`：日期，格式为 `YYYYMMDD`。

文件名会清洗 Windows/macOS/Linux 常见非法字符，并在重名时追加 `_2`、`_3` 等后缀。

### 6.12 平台尺寸预设

当前内置常用预设：

- 通用：400x400、800x800、1080x1080 方图。
- 通用：750x1000、1080x1920 竖图。
- 通用：1200x800、1920x1080 横图。
- 淘宝：800x800 主图、790x1000 详情图。
- 京东：800x800 主图。
- 拼多多：800x800 主图。

选择预设后会开启批量统一尺寸，并填充对应宽高。

## 8. 数据与隐私

当前项目没有后端服务、登录账号、远程存储或接口请求。图片读取、图层编辑、选区生成、文字绘制、模板保存和导出都在浏览器端完成。

本地持久化范围：

- 裁剪模板保存在浏览器 `localStorage`。

不会持久化：

- 当前图片。
- 当前图层状态。
- 当前裁剪区域。
- 当前文字标注。
- 当前历史记录。

刷新页面或关闭页面后，除模板外的编辑状态都会丢失。

## 9. 当前实现边界

- 产品主要面向现代桌面浏览器，未做移动端触控适配。
- 当前没有项目文件保存/打开功能。
- 多图片仍按“多图层合成”处理，不是多任务队列。
- 区域和文字是全局画布对象，不绑定到某个图层。
- 画布渲染会将活动图层置顶显示，导出则按图层数组顺序合成，多图层遮挡时预览和导出可能存在层级差异。
- 上传按钮导入图片后没有显式调用 `fitToCanvas()`；拖拽导入会自动适配画布。
- 模板套用当前实现使用模板保存时的基准宽高计算输出尺寸，尚未完全按目标活动图层尺寸缩放，跨分辨率模板复用可能不符合预期。
- 单区域导出不使用批量文件命名规则，仍使用区域名。
- 模板只保存区域布局，不保存导出配置、文字或图层。
- 历史记录保存图层工作画布 ImageData，连续大图编辑仍有内存压力。
- 魔棒在大图或复杂纹理上可能造成主线程卡顿。
- 平台预设是常用尺寸清单，不保证覆盖平台最新规则。
- 当前 UI 图标体系仍混用字符、文本和 emoji，专业化一致性还有提升空间。

## 10. 项目审核结论

### 10.1 总体结论

项目已从“多图层素材裁剪工具”进一步收敛到“电商运营批量切图工作台”。当前代码已经落地文件命名规则、固定尺寸批量导出、网格生成、等距复制、长图切片、参考线切片、模板复用、平台尺寸预设、区域预览和区域变换，形成了较完整的批量切图闭环。

与 V1.1 文档相比，若干旧风险已经被当前实现修复或缓解：

- 图层缩放下的导出坐标已按 `scaleX/scaleY` 转换。
- 魔棒已处理图层偏移与缩放。
- 画笔和橡皮已处理图层偏移与缩放。
- 批量导出数量文案已按勾选数或全部区域数显示。
- 删除/清空区域已先写入历史快照。
- 图层已支持重命名和排序。
- 批量导出已支持统一尺寸、适配模式和文件命名规则。

当前更值得优先处理的问题，是图层显示/导出层级一致性、模板跨尺寸套用、上传后画布适配、单区域命名一致性和历史记录内存策略。

### 10.2 已验证事项

- `npm run build` 通过。
- 项目仍为纯前端实现，没有发现后端接口、鉴权逻辑或明显数据外传流程。
- 依赖较少，主要是 Vue、Pinia、JSZip、file-saver。
- 产品主链路清晰：导入图片 -> 生成/编辑区域 -> 预览 -> 设置尺寸与命名 -> ZIP 导出。

### 10.3 主要问题与风险

#### P1：画布图层显示顺序与导出合成顺序可能不一致

位置：`src/composables/useCanvasEngine.ts`、`src/composables/useExport.ts`

画布渲染时会将活动图层绘制到最上层，便于用户移动和缩放当前图层；导出合成时则按 `layers` 数组顺序绘制可见图层。若多图层之间有重叠，且活动图层不是导出顺序中的最上层，用户看到的画布效果可能与导出图片不一致。

建议：统一定义图层层级语义。更稳的做法是画布预览和导出都按同一层级顺序绘制，再通过独立描边和控制点突出活动图层，而不是改变活动图层的实际绘制层级。

#### P1：模板套用未按目标图层尺寸缩放

位置：`src/composables/useCropTemplates.ts`

模板保存时使用了区域相对活动图层的比例，但套用时输出尺寸仍使用模板保存时的 `baseRect.width/height`，而不是目标活动图层的显示宽高。结果是同版式但不同分辨率图片套用模板时，可能无法按目标图层比例缩放。

建议：套用模板时使用 `targetRect.width/height` 计算 `x/y/width/height` 和多边形点位。

#### P2：上传按钮导入后未自动适配画布

位置：`src/App.vue`

拖拽导入会调用 `engine.fitToCanvas()`，但左侧上传按钮只执行 `editor.addLayer(img)`。首图或新增图层后，用户可能需要手动缩放或平移才能看到合适视图。

建议：上传入口统一走 CanvasWorkspace 暴露的添加图片方法，或在新增活动图层后触发一次 `fitToCanvas()`。

#### P2：历史记录仍有大图内存压力

位置：`src/stores/history.ts`

历史快照会保存图层工作画布的完整 ImageData。当前最大栈已降到 20，但在多张大图、连续画笔/橡皮或频繁图层操作下仍可能占用较多内存。

建议：中期引入操作级 history、图层脏区快照或画笔拖拽合并策略。

#### P2：单区域导出与批量命名规则不一致

位置：`src/composables/useExport.ts`、`src/components/RightSidebar.vue`

批量导出使用命名规则，单区域导出仍直接使用区域名称。这不影响核心流程，但在“交付素材命名一致性”上会产生差异。

建议：单区域导出也支持可选命名规则，至少复用文件名清洗逻辑。

#### P3：批量预览通过 DOM 查询触发导出

位置：`src/components/PreviewModal.vue`

批量预览弹窗通过 `document.querySelector('.export-btn')` 触发右侧栏导出按钮。当前可用，但组件耦合较强。

建议：将批量导出封装为 composable 方法或通过事件回调传入弹窗。

#### P3：UI 专业一致性仍可优化

位置：`src/components/LeftSidebar.vue`、`src/components/TopBar.vue`、`src/components/RightSidebar.vue`

当前图标混用字符、emoji 和文字按钮。作为 MVP 可接受，但若要强化专业工具感，应统一图标组件、按钮尺寸、tooltip 和可访问性。

## 11. 非功能性要求

### 11.1 性能

- 常见 2K-4K 图片应保持基本流畅。
- 超大图片、多图层、连续画笔和魔棒操作需要性能提示。
- 历史记录应避免每次操作长期保存完整像素快照。
- 批量预览和批量导出应避免长时间阻塞主线程。

### 11.2 兼容性

- 面向现代桌面浏览器。
- 依赖 Canvas、FileReader、Blob、ResizeObserver、localStorage、download 等能力。
- 暂不承诺移动端触摸体验。

### 11.3 可用性

- 上传或拖拽图片后应自动聚焦可编辑区域。
- 图层移动/缩放后，画布预览、画笔、魔棒和导出结果应保持一致。
- 清空、删除、隐藏、排序、批量生成等破坏性操作应支持撤销。
- 批量导出数量、预览数量和实际导出数量应保持一致。
- 文件命名预览应能提前暴露未知变量和重名处理结果。

## 12. 验收标准

- 上传 PNG/JPEG/WebP 后生成图层并正常显示。
- 多次上传可形成多图层，并可切换、重命名、排序、隐藏、删除、移动、缩放图层。
- 可创建并编辑所有预设形状区域。
- 可创建、完成、取消和编辑自定义多边形区域。
- 可通过魔棒生成可编辑区域。
- 可添加、移动、缩放和编辑文字。
- 可 Shift 多选、框选、批量移动、批量删除、批量变换区域。
- 可通过网格生成多个区域。
- 可基于选中区域等距复制。
- 可基于活动图层生成长图切片。
- 可通过参考线生成切片。
- 可保存、重命名、删除和套用本地模板。
- 可单独预览和导出区域。
- 可批量预览全部或勾选区域。
- 可批量导出全部或勾选区域为 ZIP。
- 批量导出可使用统一尺寸、适配方式、背景色和平台预设。
- 批量导出文件名按规则生成、清洗非法字符并自动去重。
- 撤销/重做覆盖区域、文字、图层和关键批量操作。
- `npm run build` 通过。

## 13. 后续优化优先级

### 13.1 必须优先修复

- 修复模板套用按目标图层尺寸缩放的问题。
- 统一画布预览和导出时的图层合成顺序。
- 上传按钮导入后自动适配画布。
- 单区域导出复用文件名清洗，必要时支持命名规则。

### 13.2 短期优化

- 限制或提示超大图片。
- 降低历史记录内存占用。
- 将批量预览弹窗的导出触发改为事件或 composable。
- 为模板增加导入/导出能力，便于换浏览器或换设备。
- 统一右侧面板中批量切图、模板、导出区域的交互文案。

### 13.3 中期优化

- 项目保存/打开。
- 多图片独立任务队列。
- 模板分类、搜索和导入导出。
- 魔棒预览、追加/减去选区和平滑参数。
- 对齐线、吸附和参考线管理面板。
- 统一图标体系和工具栏交互。

## 15. 最终建议

当前阶段已经具备电商批量切图 MVP 的主要闭环，不建议立即扩展大型功能。更稳的路线是先修复模板跨尺寸套用和上传适配这类一致性问题，再围绕真实运营工作流补强项目保存、模板导入导出、多图片任务队列和更专业的 UI 体系。
