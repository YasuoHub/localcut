## 1. 产品概述

LocalCut Pro 是一款本地优先的电商批量切图与素材交付工作台。用户可以导入图片，在画布中创建或批量生成裁剪区域，通过模板复用、统一尺寸、命名规则和批量预览，最终导出可交付的图片 ZIP。

产品核心价值是：**在不上传图片、不依赖后端的前提下，帮助电商运营和内容制作人员快速、稳定、可复用地完成批量素材切图交付。**

当前产品已经具备 MVP 闭环，但下一阶段的重点不应继续堆叠泛图片编辑能力，而应围绕“批量效率、交付确定性、模板复用、本地隐私”打造付费价值。

## 2. 产品定位

当前定位：

> 本地电商批量切图与素材裁剪工具，介于截图裁剪工具、批处理工具和轻量图片编辑器之间。

应强化的方向：

- 批量生成裁剪区域。
- 批量预览和批量导出。
- 统一尺寸和命名规则。
- 模板保存、迁移和复用。
- 多图片同版式处理。
- 本地项目保存和恢复。

应避免的方向：

- 不做 Photoshop、Photopea、Canva 类完整图片编辑器。
- 不做云端素材库、多人协作或模板市场的早期版本。
- 不以 AI 抠图、超分、复杂形状作为主产品叙事。
- 不把平台规则包装成实时官方规则，平台尺寸应作为可维护预设。
- 不优先做移动端触控体验。

## 3. 目标用户

### 3.1 主用户

电商运营、店铺美工、内容运营和小团队设计助理。

高频任务：

- 从商品主图、详情长图、组合图、素材拼图中批量导出上传素材。
- 将多个区域统一输出为 800x800、1080x1080、750x1000 等规格。
- 对同版式图片反复套用切图布局和命名规则。
- 交付前预览所有裁剪结果，减少上传后返工。

### 3.2 次级用户

设计与内容制作人员、游戏素材切片用户、教学资料整理用户和本地隐私敏感用户。

这些用户可以使用现有批量切图能力，但产品设计优先级仍应服务电商批量交付场景。

## 4. 核心使用流程

当前单工作区流程：

1. 用户上传、菜单导入或拖拽导入图片。
2. 系统将图片作为图层加入画布，新图层成为活动图层。
3. 用户创建裁剪区域，或用网格、等距复制、长图切片、参考线切片批量生成区域。
4. 用户编辑区域位置、尺寸、形状、名称和多选状态。
5. 用户可添加文字，或用画笔、橡皮、魔棒、智能抠图处理素材。
6. 用户设置导出格式、质量、DPR、统一尺寸、适配方式、背景色和命名规则。
7. 用户预览单个或多个区域。
8. 用户导出当前区域，或批量导出全部/勾选区域为 ZIP。

目标付费流程：

1. 用户打开项目或导入多张同版式图片。
2. 用户选择模板，自动生成区域和导出配置。
3. 用户逐张检查或微调。
4. 系统执行导出前体检。
5. 用户一键导出按图片分文件夹组织的 ZIP，并获得导出报告。

## 5. 信息架构

### 5.1 顶部栏

- 品牌：LocalCut Pro。
- 系统设置：限制裁剪框在图片内、导入图层排列方向和间距。
- 快捷键说明：撤销/重做、复制/粘贴、删除、多选、微移、图层拖动、画布缩放、参考线操作等。

规划增强：

- 文件菜单增加项目保存/打开。
- 最近项目入口。
- 导出前体检入口或状态提示。

### 5.2 左侧工具栏

当前能力：

- 图片上传。
- 智能抠图入口。
- 工具列表：选择、画笔、橡皮、矩形、圆角矩形、圆形、三角形、菱形、星形、心形、多边形、魔棒、文字。
- 矩形工具下的常用裁剪尺寸。
- 文字列表。

产品判断：

- 左侧工具已经偏“图片编辑器”，下一阶段不建议继续增加绘制工具。
- 星形、心形等非电商高频形状可保留，但不应占据核心叙事。
- 新增能力应优先进入模板、批量、导出和项目流程，而不是工具栏。

### 5.3 中央画布

当前能力：

- 多图层显示。
- 图层外框、活动图层控制点和图层名称。
- 区域遮罩、区域边框、名称和控制点。
- 文字框显示与编辑。
- 顶部和左侧标尺。
- Ctrl/Command + 点击标尺添加参考线。
- Ctrl/Command + 右键参考线删除参考线。
- 原图开关。
- 滚轮缩放、Space 拖拽或鼠标中键平移。

已具备并需保持：

- 画布预览、批量预览、单区域导出和批量导出的合成顺序必须一致。
- 活动图层高亮不能改变实际渲染层级。

当前代码中，画布绘制和导出合成都按 `layers` 的反向顺序绘制，使图层列表顶部成为视觉最上层；活动图层只额外绘制外框和控制点，不会改变图片内容的实际层级。

必须强化：

- 上传、拖拽、菜单导入后的视图适配应一致。

### 5.4 右侧面板

当前能力：

- 图层列表：切换、显示/隐藏、重命名、排序、删除。
- 工具参数：画笔、橡皮、魔棒。
- 区域属性：名称、坐标、尺寸、形状、预览、导出。
- 文字属性：内容、字号、字重、颜色。
- 区域变换：翻转、90 度旋转、自定义角度旋转。
- 批量切图：网格生成、等距复制、长图切片、参考线切片。
- 模板：保存、套用、重命名、删除。
- 导出设置：格式、质量、DPR、AI 超分、锐化。
- 批量输出尺寸：平台尺寸预设、统一尺寸、适配方式、背景色。
- 文件命名：命名规则、变量插入、校验和预览。
- 区域列表：勾选、删除、清空、批量预览、批量导出。

规划调整：

- 模板模块应升级为“生产资产管理”，支持导入/导出、分类、搜索和携带导出配置。
- 导出模块应增加导出前体检。
- 多图片队列上线后，右侧应支持任务列表或单独任务面板，避免继续堆高当前右侧栏。

## 6. 当前功能说明

### 6.1 图片与图层

当前支持多次上传或拖拽导入图片。每张图片会创建一个图层，包含名称、原始图片、工作画布、位置、缩放比例和可见性。

图层支持：

- 点击图层列表切换活动图层。
- 双击名称重命名。
- 显示或隐藏。
- 上下排序。
- 删除。
- 画布中拖拽移动。
- 控制点缩放。
- 非选择工具下 Ctrl/Command + 拖拽快速移动图层。

当前风险：

- 多图片目前仍是多图层合成，不是独立任务队列。
- 区域和文字是全局画布对象，不绑定某个图层。
- 未来改造预览、导出或多图片任务队列时，需要保留当前图层层级一致性。

### 6.2 裁剪区域

支持形状：

- 矩形。
- 圆角矩形。
- 圆形。
- 三角形。
- 菱形。
- 星形。
- 心形。
- 自定义多边形。
- 魔棒生成的自定义多边形。

区域支持：

- 画布点击选中。
- 右侧列表选中。
- Shift + 点击多选。
- Shift + 拖拽框选。
- 移动和控制点缩放。
- 精确编辑 x/y/宽/高。
- 重命名。
- 删除和清空。
- 复制和粘贴。
- 方向键微移。
- 多选后批量移动、删除和变换。

产品判断：

- 电商批量切图的高频形状是矩形、圆角矩形和自定义多边形。
- 其他装饰形状可保留，但不应继续扩展。

### 6.3 魔棒智能选区

魔棒基于活动图层工作画布执行颜色连通区域识别，并生成可编辑多边形区域。当前已处理图层偏移和缩放。

适用场景：

- 背景颜色明确。
- 素材边界清晰。
- 需要快速生成不规则裁剪区域。

限制：

- 复杂纹理、渐变、半透明边缘和抗锯齿边界仍可能需要手工调整。
- 大图或复杂区域可能带来性能压力。

### 6.4 画笔、橡皮与文字

画笔和橡皮作用于活动图层工作画布，当前已处理图层偏移和缩放，编辑结果参与导出。

文字是全局画布对象，不绑定具体图层。导出区域时，如果文字框与区域重叠，会绘制到区域内部。

产品判断：

- 这些功能作为轻量修补和标注是有价值的。
- 不建议继续向完整排版、滤镜、图像编辑方向扩展。

### 6.5 批量切图

当前批量切图能力是产品核心。

网格生成：

- 行数、列数。
- 起始 X/Y。
- 单元宽高。
- 横向和纵向间距。
- 命名前缀。
- 圆角。
- 替换现有区域或追加生成。

等距复制：

- 基于选中区域横向、纵向或自定义偏移复制。
- 多边形会同步平移顶点。

长图切片：

- 基于活动图层显示矩形生成纵向切片。
- 支持开始 Y、结束 Y、切片高度、重叠和命名前缀。

参考线切片：

- 根据活动图层边界和参考线交叉区域生成裁剪框。

下一步重点：

- 参考线管理和吸附。
- 导出前检查空区域、超出图层和源像素不足。
- 多图片任务队列中复用这些生成逻辑。

### 6.6 裁剪模板

当前模板支持：

- 保存当前区域为本地模板。
- 模板保存在 `localStorage`。
- 套用到当前活动图层。
- 重命名。
- 删除。

当前代码状态：

- 模板保存区域相对活动图层的比例。
- 套用模板时已使用目标活动图层显示矩形换算区域位置和尺寸。
- 模板当前只保存区域布局，不保存导出格式、命名规则、文字或图层。

下一步重点：

- 模板导入/导出。
- 模板携带导出配置。
- 模板分类、搜索、最近使用和收藏。

### 6.7 预览

当前支持：

- 单区域预览和放大查看。
- 批量缩略预览。
- 批量预览中进入放大查看。
- 批量预览内触发批量导出。

当前风险：

- 批量预览触发导出仍通过 DOM 查询按钮，组件耦合较强。

下一步重点：

- 批量导出流程应封装为明确方法或事件回调。
- 批量预览应显示导出前体检结果。

### 6.8 导出

支持格式：

- PNG。
- JPEG。
- WebP。

支持能力：

- 质量设置。
- DPR。
- 智能 DPR 限制。
- AI 超分。
- 锐化。
- 批量统一尺寸。
- 适配方式：裁切填满、完整包含、拉伸、原始。
- `contain` 模式背景色。
- 平台尺寸预设。
- 命名规则和去重。
- ZIP 批量导出。

当前问题：

- 单区域导出仍使用区域名，未完全复用批量命名规则。
- ZIP 默认名仍是 `output.zip`，未来应支持项目名或活动图层名。
- 需要增加导出前体检。

### 6.9 文件命名规则

当前默认规则：

```text
{imageName}_{regionName}_{index:3}
```

当前支持变量：

- `{imageName}`。
- `{regionName}`。
- `{index}`。
- `{index:2}`、`{index:3}`。
- `{width}`。
- `{height}`。
- `{format}`。
- `{date}`。

当前支持：

- 未知变量提示。
- 文件名前 3 项预览。
- 非法字符清洗。
- 重名自动追加后缀。

下一步重点：

- 单区域导出复用该规则。
- 多图片队列新增 `{jobName}`、`{jobIndex}`。
- 导出报告记录命名结果。

### 6.10 平台尺寸预设

当前内置：

- 通用方图、竖图、横图。
- 淘宝主图和详情图。
- 京东主图。
- 拼多多主图。

产品判断：

- 平台规则会变化，且不同店铺有自己的要求。
- 当前预设应定位为“常用尺寸”，不应承诺覆盖平台最新官方规则。

下一步重点：

- 用户自定义规格包。
- 规格导入/导出。
- 选择规格只填充尺寸，不强制覆盖命名规则。

### 6.11 智能抠图、超分与锐化

当前支持：

- 本地 ONNX 模型抠图。
- 手动遮罩编辑。
- 边缘羽化、扩展、收缩。
- 抠图结果可发送到主画布。
- 导出时可启用 AI 超分和锐化。

产品判断：

- 这些能力可以增强素材质量。
- 它们不应成为主定位，否则产品会被拉向泛 AI 修图工具。
- 更好的设计是把它们包装成“导出增强预设”，服务于批量交付。

## 7. 数据与隐私

当前项目没有后端服务、登录账号、远程存储或业务接口。图片读取、图层编辑、选区生成、文字绘制、模板保存、AI 抠图和导出都在本地完成。

当前持久化：

- 裁剪模板保存在浏览器 `localStorage`。
- 模型可通过 IndexedDB 缓存。

当前不会持久化：

- 当前项目。
- 当前图片。
- 当前图层状态。
- 当前裁剪区域。
- 当前文字标注。
- 当前历史记录。

规划：

- 支持 `.localcut` 项目文件。
- Electron 版本优先支持本地文件保存与打开。
- 浏览器版本可通过下载/上传项目 JSON 实现。

## 8. 当前实现边界

- 产品主要面向桌面浏览器和 Electron 桌面应用。
- 当前没有项目保存/打开。
- 当前没有多图片独立任务队列。
- 多图片仍按多图层工作区处理。
- 区域和文字尚未绑定到具体图层或任务。
- 模板当前不保存导出配置。
- 单区域导出与批量命名规则不完全一致。
- 历史记录会保存图层工作画布 ImageData，大图连续编辑仍有内存压力。
- 魔棒、锐化、大尺寸导出仍可能造成性能压力。
- UI 图标体系仍混用字符、emoji 和文字按钮，专业一致性有提升空间。

## 9. 产品审核结论

### 9.1 总体结论

项目方向没有走偏，但已经处在容易走偏的位置。

正确方向是继续强化：

- 批量切图。
- 批量导出。
- 模板复用。
- 本地隐私。
- 交付稳定性。

需要克制：

- 不继续扩展泛图片编辑工具。
- 不把 AI 能力推到核心定位前面。
- 不过早做云端协作和模板市场。
- 不把多图片简单堆成多图层，而是要做任务队列。

### 9.2 已验证事项

- `npm run build` 通过。
- 当前主要依赖为 Vue、Pinia、JSZip、file-saver、onnxruntime-web。
- 未发现后端业务接口或明显数据外传流程。
- 核心链路清晰：导入图片 -> 生成/编辑区域 -> 预览 -> 设置尺寸与命名 -> ZIP 导出。
- 模板套用逻辑当前已按目标图层显示矩形换算比例。

### 9.3 当前最重要问题

P0：

- 上传入口、拖拽导入、菜单导入的视图适配必须一致。
- 单区域导出应复用命名规则和文件名清洗。
- 批量预览触发导出的组件耦合需要解开。

P1：

- 增加导出前体检。
- 降低历史记录内存压力。
- 模板导入/导出。
- 模板携带导出配置。

P2：

- 项目保存/打开。
- 最近项目和自动恢复。
- 多图片任务队列。
- 平台规格包自定义。

## 10. 付费价值设计

首个 Pro 版本应围绕“高频批量生产”收费，而不是围绕“更多工具”收费。

建议免费层：

- 单张图片处理。
- 基础区域创建。
- 少量区域批量导出。
- 基础统一尺寸。
- 少量本地模板。
- 基础命名规则。

建议 Pro 层：

- 不限区域批量导出。
- 模板导入/导出。
- 模板保存导出配置。
- 项目保存/打开。
- 多图片任务队列。
- 高级命名变量。
- 导出前体检。
- 导出报告。
- AI 抠图、超分、锐化作为增强能力。

核心付费理由：

- 省时间。
- 少返工。
- 可复用。
- 可迁移。
- 本地隐私。

### V1.3：可靠性交付版

目标：让现有 MVP 可信。

关键指标：

- 每批平均导出图片数。
- 模板保存率。
- 模板复用率。
- 命名规则使用率。

## 13. 验收标准

当前 MVP 应满足：

- 上传 PNG/JPEG/WebP 后生成图层并正常显示。
- 多图层可切换、重命名、排序、隐藏、删除、移动、缩放。
- 可创建和编辑核心裁剪区域。
- 可通过网格、等距复制、长图切片、参考线切片批量生成区域。
- 可保存、重命名、删除和套用本地模板。
- 可单独预览和批量预览。
- 可单独导出和批量导出 ZIP。
- 批量导出可使用统一尺寸、适配方式、背景色和平台预设。
- 批量导出文件名按规则生成、清洗非法字符并自动去重。
- 撤销/重做覆盖区域、文字、图层和关键批量操作。
- `npm run build` 通过。

下一阶段验收：

- 画布、预览、单导出、批导出的图层顺序继续保持一致。
- 所有导入入口都自动适配画布。
- 单区域导出和批量导出命名规则一致。
- 导出前能发现命名、尺寸、空区域和源像素不足问题。
- 模板可以导出到文件并在另一环境导入。
- 项目关闭后可以再次打开恢复工作区。
- 多图片队列中每张图可以独立套模板、预览和导出。

## 15. 最终建议

LocalCut Pro 当前已经具备电商批量切图 MVP 的主要闭环。下一阶段应停止横向扩工具，把开发重心转向可靠性交付、模板资产、项目保存和多图片任务队列。

一句话原则：

> 让用户更快、更稳、更可复用地交付一批图片，这就是产品的核心竞争力和付费价值。
