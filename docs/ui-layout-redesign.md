# LocalCut Pro UI 功能布局重设计说明

版本：V1.2
日期：2026-06-09
目标：基于现有功能重新梳理 UI 信息架构，形成符合“本地批量切图与素材交付工作台”定位的生产工具界面，并避免同一功能在多个位置重复展示。

当前开发目标：P0、P1 的核心能力已经基本具备，且当前代码中已经出现左侧窄工具栏、右侧 Tabs、底部交付栏等关键布局形态。本阶段重点是**明确可销售的产品目标、稳定专业工具心智、降低 Figma/Photoshop 用户迁移成本**，不是继续扩展泛图片编辑功能。

## 0. 产品目标定稿

LocalCut Pro 不应被设计成“轻量 Photoshop”或“另一个在线设计工具”，而应被设计成一个用户愿意为高频交付效率付费的本地切图工作台。

一句话目标：

> 让已经会用 Figma、Photoshop 或类似设计工具的人，用更少步骤、更少学习成本，把一批图片稳定切成可交付文件。

### 0.1 可销售定位

产品对外定位建议为：

> 本地优先的批量切图与图片交付工具，面向电商运营、设计助理、内容团队和需要从 Figma/PS 流程迁移出来的高频切图用户。

用户购买的不是“更多形状”“更多 AI 按钮”或“更多修图能力”，而是：

- 更快：同样一批图，少点几次、少重复设置。
- 更稳：预览、命名、尺寸和最终导出一致。
- 可复用：模板可以保存布局和导出配置，换图后继续用。
- 可迁移：会 Figma/PS 的用户能自然理解画布、对象、属性、模板和导出。
- 本地安全：图片、模板和导出都尽量在本地完成，不把商品素材上传到第三方。

### 0.2 核心用户与主场景

主用户：

- 电商运营、店铺美工、内容运营、设计助理。
- 熟悉 Figma、Photoshop、Canva、Photopea 或截图裁剪工具，但不想为了重复切图打开重型设计软件。
- 高频处理商品主图、详情长图、拼图素材、社媒封面、课程图和素材包。

主场景：

```text
导入图片 -> 画布上框选/批量生成切图区域 -> 套模板 -> 设置导出规格和命名 -> 体检/预览 -> 导出 ZIP
```

这个流程必须比 Figma/PS 的“复制画板、手动切片、逐个导出、手动改名”更短、更稳定。

### 0.3 Figma/PS 用户迁移原则

迁移设计要借用用户已熟悉的概念，但不要照搬专业设计软件的复杂度。

应借用：

- 画布：用户在中间确认图片、区域和视觉结果。
- 对象：图层、裁剪框、文字框都可选中、重命名、排序或删除。
- 属性：选中对象后在右侧改名称、位置、尺寸、样式和变换。
- 模板：类似组件/动作/预设，但服务于切图布局和导出配置。
- 导出：像 Export Panel 一样统一管理格式、尺寸、命名和质量。

不应照搬：

- 不做无限工具栏和复杂菜单树。
- 不做完整图层混合、滤镜、蒙版、路径编辑、排版系统。
- 不把每个工具参数都摊在左侧或顶部。
- 不让用户理解过多图像工程参数，例如 DPR、USM、超分模型细节；高级参数可以折叠。

### 0.4 简单操作的设计标准

一个功能只有在满足以下标准时，才值得进入主界面：

- 第一次使用时不用读说明也能猜到入口。
- 高频动作最多 2 步内可完成。
- 导出前能看懂会得到多少文件、什么尺寸、什么文件名。
- 出错时能定位到具体区域或设置项。
- 迁移用户能用熟悉心智理解它：画布、对象、属性、模板、导出。

### 0.5 首个可收费体验边界

首个可收费版本不必等到所有路线图完成，但必须让用户感受到“这是生产工具，不是玩具”。

最低可收费体验建议包含：

- 不限数量的批量导出。
- 模板导入/导出。
- 模板携带导出配置。
- 导出前体检和问题定位。
- 高级命名规则。
- 项目保存/打开，或至少可靠的本地工作恢复。

多图片任务队列是更强付费点，但可以作为后续 Pro 主版本推出；在它之前，当前单图/多区域工作台必须足够可靠。

## 1. 设计原则

LocalCut Pro 的 UI 应服务一个主目标：让用户更快、更稳、更可复用地交付一批图片。

界面不应追求“图片编辑器功能大全”的展示感，而应像电商素材生产台：

- 画布优先：中间区域始终最大化，用户主要在画布上确认图片和裁剪结果。
- 批量优先：批量生成、模板复用、批量预览、批量导出是主流程。
- 交付优先：尺寸、命名、导出检查和 ZIP 导出必须清晰、固定、可预测。
- 状态优先：用户应随时知道当前图片、当前选择、选中区域数、导出是否有问题。
- 克制编辑：画笔、橡皮、魔棒、文字、抠图是辅助工具，不应压过批量交付主流程。
- 不重复展示：一个功能只保留一个主操作入口，其他位置只显示状态、摘要或跳转。

## 2. 功能归属规则

### 2.1 去重原则

同一个功能可以有一个主入口和必要的上下文快捷入口，但不能在多个模块中以完整表单重复出现。

规则：

- 完整配置只放在一个模块中。
- 其他位置只允许显示摘要、状态、快捷跳转或只读信息。
- 如果一个功能既能单区使用又能批量使用，默认归入批量交付流程，单区只保留轻量动作。
- 全局命令放顶部，画布工具放左侧，选中对象编辑放右侧，交付动作放底部固定栏。

### 2.2 功能归属表

| 功能 | 主展示位置 | 其他位置允许显示 | 不允许重复 |
|---|---|---|---|
| 导入图片 | 左侧顶部按钮、空画布拖拽区 | 菜单栏快捷命令 | 左侧不再放完整上传说明 |
| 图层管理 | 右侧“对象”Tab | 顶部显示当前活动图层名 | 画布上不提供完整图层列表 |
| 选择/绘制工具 | 左侧工具栏 | 顶部状态显示当前工具 | 右侧不重复工具按钮 |
| 工具参数 | 右侧“属性”Tab | 左侧工具 tooltip 显示快捷提示 | 左侧不展开参数表单 |
| 区域属性 | 右侧“属性”Tab | 画布标签显示名称/尺寸 | 区域列表不重复坐标输入 |
| 文字属性 | 右侧“属性”Tab | 画布显示文字框 | 左侧文字列表取消或改到对象列表 |
| 批量生成 | 右侧“批量”Tab | 画布参考线状态栏 | 顶部不放网格/切片表单 |
| 模板 | 右侧“模板”Tab | 顶部显示当前套用模板名 | 批量 Tab 不重复模板列表 |
| 导出格式/尺寸/命名 | 右侧“导出”Tab | 底部导出栏显示摘要 | 区域属性里不重复完整导出设置 |
| 区域/文字卡片列表 | 右侧底部可拖拽卡片列表面板 | 底部导出栏显示导出数量 | 画布不承担勾选清单 |
| 批量预览 | 底部导出栏主按钮 | 区域清单内单行预览 | 右侧多个 Tab 不重复批量预览按钮 |
| 批量导出 | 底部固定导出栏 | 批量预览弹窗内可确认导出 | 右侧滚动内容中不再放另一个导出按钮 |
| 智能抠图 | 左侧工具栏入口，独立模态工作区 | 右侧属性显示当前抠图结果摘要 | 不放进导出设置作为主功能 |
| AI 超分/锐化 | 右侧“导出”Tab 的增强设置 | 导出栏显示增强状态 | 抠图模块不重复超分设置 |
| 参考线 | 画布标尺交互，右侧“批量”Tab 管理 | 状态栏显示参考线数量 | 顶部不重复参考线按钮组 |

## 3. 全局布局

推荐布局：

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 顶部命令栏：品牌 / 设置 / 快捷键                                             │
├──────┬───────────────────────────────────────────────────────┬───────────────┤
│ 左侧 │ 中央画布工作区                                         │ 右侧任务面板  │
│ 工具 │ 标尺 + 画布 + 参考线 + 区域 + 图层控制点                │ 属性/批量/模板│
│ 82px │ 底部画布状态栏                                         │ 导出/清单     │
├──────┴───────────────────────────────────────────────────────┴───────────────┤
│ 底部固定交付栏：已选数量 / 导出检查 / 批量预览 / 导出 ZIP                     │
└──────────────────────────────────────────────────────────────────────────────┘
```

布局尺寸：

- 顶部命令栏：44-48px。
- 左侧工具栏：约 82px，采用上图标、下文字的上下结构，避免纯图标难以识别。
- 右侧任务面板：320-360px，可折叠，最小 280px。
- 底部交付栏：48-56px，始终固定。
- 画布状态栏：24-28px，附着在画布底部内侧或画布容器底部。

视觉风格：

- 背景使用石墨黑和暖灰，不使用大面积渐变。
- 画布区比面板更暗，面板边界清晰但低对比。
- 主操作使用绿色，选中和参考线使用蓝色，警告使用琥珀色，危险操作使用红色。
- 图标统一使用线性图标，禁止 emoji 作为正式图标。
- 面板内使用紧凑分组，不做大卡片堆叠。

## 4. 顶部命令栏

### 4.1 功能定位

顶部栏只承载项目级和全局级命令，不放局部参数。

包含：

- 品牌：LocalCut Pro。
- 设置和快捷键。
- 系统设置：限制裁剪框、图层名称展示、导入图层排列方向和间距。
- 后续项目能力完成后，再加入项目级状态和文件命令。

不包含：

- 导入图片主入口。当前主入口在左侧和空画布。
- 编辑命令常驻按钮。当前通过快捷键和 Electron 菜单命令处理。
- 裁剪形状按钮。
- 批量生成参数。
- 模板列表。
- 具体导出尺寸输入。

### 4.2 状态描述

空项目状态：

- 品牌左侧正常显示。
- 导入图片可用。
- 新建、打开项目、保存项目在项目文件能力完成前不展示，或以禁用状态放入更多菜单。
- 不显示图层名、图层数量、区域数量、勾选数量或告警数量。

已导入图片状态：

- 顶部仍只显示品牌、设置和快捷键。
- 项目保存能力完成前，不显示可点击保存按钮。

有未保存修改状态：

- 文件名后显示小圆点或 `未保存` 标签。
- 关闭或新建时弹出保存确认。

导出检查正常：

- 显示绿色状态点和“可导出”。
- 点击可打开导出检查详情。

导出检查警告：

- 显示琥珀色状态点和 `3 个警告`。
- 警告不阻止导出，但导出前需要用户可见。

导出检查错误：

- 显示红色状态点和 `无法导出`。
- 底部导出按钮禁用，点击状态打开错误列表。

### 4.3 当前实现基线

当前 `TopBar.vue` 主要包含品牌、设置和快捷键，符合“顶部不放局部参数”的方向。

已实现并需要保持：

- 品牌常驻左侧。
- 设置弹层包含裁剪约束、图层名称展示、导入图层排列方向和间距。
- 快捷键弹层包含撤销/重做、复制/粘贴、选择、删除、微移、画布平移缩放和参考线操作。
- 不在顶部展示导出配置、批量表单或模板列表。

下一步：

- 项目保存/打开完成前，不要在顶部做可点击的假按钮。
- 导出体检摘要当前放在底部交付栏更合适，顶部最多在项目化阶段显示极短状态。

顶部栏示例：

```text
LocalCut Pro                                           设置  快捷键
```

不同状态：

- 无图层：`未导入图片 | 无可导出内容`。
- 有图层无区域：`banner.jpg | 图层 1 | 区域 0 | 暂无区域`。
- 有警告：`3 个警告` 使用琥珀色。
- 有错误：`无法导出` 使用红色。

## 5. 左侧工具栏

### 5.1 功能定位

左侧只放画布直接操作工具，不放业务配置。

推荐工具顺序：

1. 导入图片。
2. 智能抠图。
3. 选择。
4. 画笔。
5. 橡皮擦。
6. 魔棒。
7. 图形。
8. 文字。

图形按钮 hover 后在右侧展开全部形状：

- 矩形。
- 圆角矩形。
- 圆形。
- 三角形。
- 菱形。
- 星形。
- 心形。
- 多边形。

理由：

- 电商批量切图最常用的是矩形、圆角矩形、多边形和魔棒。
- 星形、心形等装饰形状不是核心，常驻会误导产品定位。

### 5.2 状态描述

默认状态：

- 所有工具显示为图标 + 文字的紧凑按钮。
- 未选中工具使用低对比线性图标。
- hover 显示 tooltip，例如“矩形区域 R”。

选中状态：

- 当前工具按钮背景变为深色高亮。
- 图标使用蓝色或绿色。
- 左侧有 2px 高亮条。
- 顶部或画布状态栏显示当前工具名。

禁用状态：

- 未导入图片时，除导入、打开项目、智能抠图上传入口外，区域/画笔类工具禁用。
- 禁用图标透明度降低，hover tooltip 显示原因：`请先导入图片`。

绘制中状态：

- 当前绘制工具保持高亮。
- 画布 cursor 切换为 crosshair。
- 画布状态栏显示当前绘制尺寸。

抠图处理中状态：

- 智能抠图按钮显示小型进度环。

## 7. 右侧任务面板

2. 批量。

- 批量预览、底部交付栏和实际 ZIP 数量一致。

### 13.3 第三阶段：模板资产化打磨

- 模板列表继续突出区域数、尺寸、适配方式、命名规则和格式摘要。
- 模板保存时明确“只保存区域”与“区域 + 导出配置”的区别。
- 导入/导出模板时给出兼容、冲突和成功反馈。
- 模板搜索、分类、收藏和最近使用要服务于 20 个以上模板的管理。

验收：

- 用户能理解模板不是装饰预设，而是可复用的生产资产。
- 换图套模板后，底部交付栏摘要和导出体检同步更新。
- 模板导入到另一台机器后仍能套用并导出。

### 13.4 第四阶段：免费/Pro 分层体验

- 免费限制优先围绕批量区域数、模板数量、模板导入/导出、项目保存和多图队列。
- 不限制单张手动裁剪、基础预览、基础导出和本地隐私能力。
- 触发限制时解释价值：例如“升级后可一次导出全部 N 项”，不要只说“不可用”。
- Pro 能力入口不要打断基础流程，优先放在交付栏、模板 Tab、导出 Tab 和项目菜单中。

验收：

- 免费用户能完整体验核心闭环。
- 高频用户自然遇到升级理由。
- 升级提示不影响正在编辑的画布和对象状态。

### 13.5 第五阶段：项目连续性与多图队列设计

- 先设计 `.localcut` 项目保存/打开，让用户能分多次完成工作。
- 再设计多图片任务队列，避免继续把多张图作为叠加图层处理。
- 多图队列必须复用模板、命名、体检和导出报告，而不是另开一套孤立流程。

验收：

- 关闭后重新打开项目，图层、区域、文字、参考线和导出配置保持一致。
- 多图导入时每张图片是独立任务，而不是普通图层堆叠。
- 同版式 20 张图可以一键套模板、逐张微调、按图分文件夹导出。

## 14. 开发约束

### 14.1 业务逻辑约束

- 不改变图层数组顺序语义：列表顶部仍是视觉最上层。
- 不改变区域选择、多选、勾选导出的语义。
- 不改变模板坐标换算逻辑。
- 不改变导出合成逻辑。
- 不改变智能抠图工作区的数据流。
- 不新增多图队列。

### 14.2 状态管理约束

- 优先复用现有 Pinia store：`editor`、`export`、`history`、`matting`。
- 右侧 Tab 当前激活状态可以先放组件本地状态。
- 如果多个组件需要打开同一个预览或导出动作，优先抽 composable，不要 DOM 查询。
- 不为纯视觉状态新增复杂全局 store。

### 14.3 样式约束

- 新增全局 UI token，减少每个组件 scoped style 重复定义。
- 卡片圆角不超过 6px，弹窗不超过 8px。
- 工具按钮、图标按钮必须有固定宽高，避免文字和图标导致布局跳动。
- 列表行高度固定，区域名过长使用省略号。
- 所有主按钮同一颜色，同一高度。
- 危险操作只用红色描边或红色文字，不用大面积红色块。

### 14.4 可访问性和效率约束

- 图标按钮必须有 `title` 或 tooltip。
- 禁用按钮必须能解释原因。
- 导出中禁止重复点击。
- 删除、清空、替换区域需要确认或可撤销。
- 高频动作不要隐藏到二级菜单：批量预览、导出、套用模板、生成区域必须一眼能找到。

## 18. 当前 UI 风格与交互定稿

基于产品目标，LocalCut Pro 当前 UI 定位定为：**本地电商批量切图与素材交付工作台**。界面应该让用户相信它适合长时间处理商品图、详情图和批量交付任务，而不是像营销页、AI 玩具或泛图片编辑器。

### 18.1 视觉风格

- 整体气质：克制、专业、高密度、偏生产工具。
- 背景体系：石墨黑、暖灰、低对比边框，避免大面积蓝紫和装饰渐变。
- 主色：绿色只用于主操作、可导出、当前工具、当前选中对象和可执行成功状态。
- 辅助色：蓝色只用于画布空间辅助，例如参考线、选区框、标尺相关反馈；不要用于面板卡片装饰。
- 警告和错误：琥珀色表示警告，红色表示阻断或危险操作。
- 圆角和阴影：面板内控件圆角保持 5-6px，弹窗可到 8px；不使用浮夸阴影。
- 卡片使用：只用于重复项、弹窗和明确成组的工具块；不把页面区域层层包成卡片。

### 18.2 信息架构

主流程固定为：

```text
导入图片 -> 生成/编辑区域 -> 套用或保存模板 -> 设置导出 -> 体检/预览 -> 导出 ZIP
```

对应布局职责：

- 顶部：品牌、设置、快捷键，后续加入项目级状态；不放局部参数。
- 左侧：只放画布直接操作工具，保持窄栏。
- 中间：画布、标尺、参考线、对象视觉确认，保留最大工作面积。
- 右侧：属性、批量、模板、导出和清单；一个完整配置只出现一次。
- 底部：交付栏固定展示导出数量、体检状态、批量预览和导出 ZIP。

### 18.3 交互规则

- 高频交付动作必须一眼可见：批量预览、导出 ZIP、套用模板、生成区域。
- 导出设置只能在导出 Tab 完整配置，其他位置只显示摘要或使用该配置执行。
- 模板 Tab 是生产资产管理区，模板列表、搜索、分类、导入导出必须处在同一资产库容器内。
- 区域/文字/图层选中态统一用绿色，和主操作一致；画布参考线继续保留蓝色以区别空间辅助。
- 未导入图片时，除了导入和独立抠图入口，其他画布编辑工具保持禁用并解释原因。
- 删除、清空、覆盖模板等破坏性动作必须可撤销或确认。
- AI 抠图、超分、锐化属于交付增强能力，不作为界面主叙事。

### 18.4 当前细节优化方向

- 去除残留蓝紫 UI：标尺、空画布、模板资产卡片和右侧列表选中态应回到石墨灰/绿色体系。
- 提高信息密度：列表行、按钮和输入框保持稳定高度，长文本截断，不让控件因内容跳动。
- 收敛卡片层级：模板资产库内部可以用细分隔线区分控制区和列表，不再拆成多个并列卡片。
- 保持底部交付栏权重：导出状态和主按钮始终固定，不被右侧 Tab 滚动影响。
- 后续图标统一时，优先替换 emoji 和字符按钮为线性图标，避免混杂视觉语言。
