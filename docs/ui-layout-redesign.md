# LocalCut Pro UI 功能布局重设计说明

版本：V1.1
日期：2026-06-08
目标：基于现有功能重新梳理 UI 信息架构，形成符合“本地电商批量切图与素材交付工作台”定位的生产工具界面，并避免同一功能在多个位置重复展示。

当前开发目标：P0、P1 的核心能力已经基本具备，本阶段重点是**布局优化、视觉统一和操作路径收敛**，不是继续扩展新业务功能。

## 1. 设计原则

LocalCut Pro 的 UI 应服务一个主目标：让用户更快、更稳、更可复用地交付一批图片。

界面不应追求“图片编辑器功能大全”的展示感，而应像电商素材生产台：

- 画布优先：中间区域始终最大化，用户主要在画布上确认图片和裁剪结果。
- 批量优先：批量生成、模板复用、批量预览、批量导出是主流程。
- 交付优先：尺寸、命名、导出检查和 ZIP 导出必须清晰、固定、可预测。
- 状态优先：用户应随时知道当前图片、当前选择、选中区域数、导出是否有问题。
- 克制编辑：画笔、橡皮、魔棒、文字、抠图是辅助工具，不应压过批量交付主流程。
- 不重复展示：一个功能只保留一个主操作入口，其他位置只显示状态、摘要或跳转。

## 2. 功能归属规则

### 2.1 去重原则

同一个功能可以有一个主入口和必要的上下文快捷入口，但不能在多个模块中以完整表单重复出现。

规则：

- 完整配置只放在一个模块中。
- 其他位置只允许显示摘要、状态、快捷跳转或只读信息。
- 如果一个功能既能单区使用又能批量使用，默认归入批量交付流程，单区只保留轻量动作。
- 全局命令放顶部，画布工具放左侧，选中对象编辑放右侧，交付动作放底部固定栏。

### 2.2 功能归属表

| 功能 | 主展示位置 | 其他位置允许显示 | 不允许重复 |
|---|---|---|---|
| 导入图片 | 左侧顶部按钮、空画布拖拽区 | 菜单栏快捷命令 | 左侧不再放完整上传说明 |
| 图层管理 | 右侧“对象”Tab | 顶部显示当前活动图层名 | 画布上不提供完整图层列表 |
| 选择/绘制工具 | 左侧工具栏 | 顶部状态显示当前工具 | 右侧不重复工具按钮 |
| 工具参数 | 右侧“属性”Tab | 左侧工具 tooltip 显示快捷提示 | 左侧不展开参数表单 |
| 区域属性 | 右侧“属性”Tab | 画布标签显示名称/尺寸 | 区域列表不重复坐标输入 |
| 文字属性 | 右侧“属性”Tab | 画布显示文字框 | 左侧文字列表取消或改到对象列表 |
| 批量生成 | 右侧“批量”Tab | 画布参考线状态栏 | 顶部不放网格/切片表单 |
| 模板 | 右侧“模板”Tab | 顶部显示当前套用模板名 | 批量 Tab 不重复模板列表 |
| 导出格式/尺寸/命名 | 右侧“导出”Tab | 底部导出栏显示摘要 | 区域属性里不重复完整导出设置 |
| 区域/文字卡片列表 | 右侧底部可拖拽卡片列表面板 | 底部导出栏显示导出数量 | 画布不承担勾选清单 |
| 批量预览 | 底部导出栏主按钮 | 区域清单内单行预览 | 右侧多个 Tab 不重复批量预览按钮 |
| 批量导出 | 底部固定导出栏 | 批量预览弹窗内可确认导出 | 右侧滚动内容中不再放另一个导出按钮 |
| 智能抠图 | 左侧工具栏入口，独立模态工作区 | 右侧属性显示当前抠图结果摘要 | 不放进导出设置作为主功能 |
| AI 超分/锐化 | 右侧“导出”Tab 的增强设置 | 导出栏显示增强状态 | 抠图模块不重复超分设置 |
| 参考线 | 画布标尺交互，右侧“批量”Tab 管理 | 状态栏显示参考线数量 | 顶部不重复参考线按钮组 |

## 3. 全局布局

推荐布局：

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 顶部命令栏：品牌 / 设置 / 快捷键                                             │
├──────┬───────────────────────────────────────────────────────┬───────────────┤
│ 左侧 │ 中央画布工作区                                         │ 右侧任务面板  │
│ 工具 │ 标尺 + 画布 + 参考线 + 区域 + 图层控制点                │ 属性/批量/模板│
│ 82px │ 底部画布状态栏                                         │ 导出/清单     │
├──────┴───────────────────────────────────────────────────────┴───────────────┤
│ 底部固定交付栏：已选数量 / 导出检查 / 批量预览 / 导出 ZIP                     │
└──────────────────────────────────────────────────────────────────────────────┘
```

布局尺寸：

- 顶部命令栏：44-48px。
- 左侧工具栏：约 82px，采用上图标、下文字的上下结构，避免纯图标难以识别。
- 右侧任务面板：320-360px，可折叠，最小 280px。
- 底部交付栏：48-56px，始终固定。
- 画布状态栏：24-28px，附着在画布底部内侧或画布容器底部。

视觉风格：

- 背景使用石墨黑和暖灰，不使用大面积渐变。
- 画布区比面板更暗，面板边界清晰但低对比。
- 主操作使用绿色，选中和参考线使用蓝色，警告使用琥珀色，危险操作使用红色。
- 图标统一使用线性图标，禁止 emoji 作为正式图标。
- 面板内使用紧凑分组，不做大卡片堆叠。

当前已经具备并需要保留：

- 上传后调用 `fitToCanvas()`。
- 导出检查能力。
- 模板保存和套用导出配置。
- 批量生成区域。
- 批量统一尺寸。
- 命名规则预览和校验。
- 智能抠图模态工作区。

本次 UI 优化不要破坏这些能力，不要为了视觉重排重新实现业务逻辑。

### 3.2 推荐组件重组

第一轮改造建议只做组件重组和样式统一。

建议新增组件：

| 新组件 | 职责 | 迁移来源 |
|---|---|---|
| `DeliveryBar.vue` | 底部固定交付栏，显示导出数量、检查状态、批量预览、导出 ZIP | `RightSidebar.vue` 底部区域列表按钮、导出状态 |
| `RightPanelTabs.vue` | 右侧 Tabs 容器，管理当前 Tab | `RightSidebar.vue` 主结构 |
| `ObjectPanel.vue` | 图层列表、文字对象列表 | `RightSidebar.vue` 图层 section、`LeftSidebar.vue` 文字列表 |
| `PropertiesPanel.vue` | 区域/文字/图层/工具参数属性 | `RightSidebar.vue` 属性、文字、画笔、橡皮、魔棒、变换 section |
| `ChecklistPanel.vue` | 区域交付清单 | `RightSidebar.vue` 底部区域列表 |
| `CanvasStatusBar.vue` | 缩放、坐标、选中数量、工具状态 | `CanvasWorkspace.vue` 新增 |

保留并复用：

| 现有组件 | 调整方式 |
|---|---|
| `BatchCutPanel.vue` | 放入“批量”Tab，不改业务逻辑，先只统一视觉 |
| `TemplatePanel.vue` | 放入“模板”Tab，保留导出配置保存/套用能力 |
| `ExportSizePanel.vue` | 放入“导出”Tab |
| `ExportNamingPanel.vue` | 放入“导出”Tab |
| `ExportInspectionPanel.vue` | 放入“导出”Tab，并把摘要同步到底部交付栏 |
| `PreviewModal.vue` | 保留，但导出触发应逐步改为事件/回调 |

### 3.3 第一轮不建议做的改动

为了降低风险，第一轮 UI 优化不要做：

- 不做多图片任务队列。
- 不重写画布引擎。
- 不重写导出逻辑。
- 不重写模板数据结构。
- 不新增复杂项目文件能力。
- 不把所有功能一次性改成全新状态管理。
- 不为了图标统一马上引入复杂设计系统。

第一轮只做：

- 面板重组。
- 操作入口去重。
- 底部交付栏固定。
- 视觉变量统一。
- 按钮、输入框、列表、状态标签统一。
- 右侧长滚动变为 Tabs。

第一轮特别注意：

- 不要把已有能力迁移成“暂时不可见”。如果某个功能还没有新组件承接，宁愿先保留在原位置或放进过渡折叠组。
- 不要做假的项目保存/打开按钮。项目文件能力未完成前，顶部只显示状态摘要、导入、设置和快捷键；项目命令可以暂不展示，或放在禁用状态的更多菜单中。
- 不要因为底部交付栏迁移而删除右侧或弹窗依赖的导出动作。批量导出必须先抽成事件、回调或 composable，再移除旧按钮。

## 4. 顶部命令栏

### 4.1 功能定位

顶部栏只承载项目级和全局级命令，不放局部参数。

包含：

- 品牌：LocalCut Pro。
- 项目命令：新建、打开项目、保存项目。注意：当前项目保存/打开尚未完全实现，第一轮不要做成可点击主按钮。
- 导入：导入图片。
- 编辑命令：撤销、重做。
- 设置和快捷键。

不包含：

- 裁剪形状按钮。
- 批量生成参数。
- 模板列表。
- 具体导出尺寸输入。

### 4.2 状态描述

空项目状态：

- 品牌左侧正常显示。
- 导入图片可用。
- 新建、打开项目、保存项目在项目文件能力完成前不展示，或以禁用状态放入更多菜单。
- 不显示图层名、图层数量、区域数量、勾选数量或告警数量。

已导入图片状态：

- 保存项目可用。
- 顶部仍只显示品牌、设置和快捷键。

有未保存修改状态：

- 文件名后显示小圆点或 `未保存` 标签。
- 关闭或新建时弹出保存确认。

导出检查正常：

- 显示绿色状态点和“可导出”。
- 点击可打开导出检查详情。

导出检查警告：

- 显示琥珀色状态点和 `3 个警告`。
- 警告不阻止导出，但导出前需要用户可见。

导出检查错误：

- 显示红色状态点和 `无法导出`。
- 底部导出按钮禁用，点击状态打开错误列表。

### 4.3 当前实现调整建议

当前 `TopBar.vue` 主要包含品牌、设置和快捷键。第一轮可以这样调整：

- 保留品牌。
- 不增加工作区摘要和导出检查摘要，避免顶部信息过载。
- 保留设置和快捷键弹层。
- 项目保存/打开如果尚未完全实现，不要做成可点击主按钮；可以暂不展示，或以禁用状态放在更多菜单中。

顶部栏示例：

```text
LocalCut Pro                                           设置  快捷键
```

不同状态：

- 无图层：`未导入图片 | 无可导出内容`。
- 有图层无区域：`banner.jpg | 图层 1 | 区域 0 | 暂无区域`。
- 有警告：`3 个警告` 使用琥珀色。
- 有错误：`无法导出` 使用红色。

## 5. 左侧工具栏

### 5.1 功能定位

左侧只放画布直接操作工具，不放业务配置。

推荐工具顺序：

1. 导入图片。
2. 智能抠图。
3. 选择。
4. 画笔。
5. 橡皮擦。
6. 魔棒。
7. 图形。
8. 文字。

图形按钮 hover 后在右侧展开全部形状：

- 矩形。
- 圆角矩形。
- 圆形。
- 三角形。
- 菱形。
- 星形。
- 心形。
- 多边形。

理由：

- 电商批量切图最常用的是矩形、圆角矩形、多边形和魔棒。
- 星形、心形等装饰形状不是核心，常驻会误导产品定位。

### 5.2 状态描述

默认状态：

- 所有工具显示为图标 + 文字的紧凑按钮。
- 未选中工具使用低对比线性图标。
- hover 显示 tooltip，例如“矩形区域 R”。

选中状态：

- 当前工具按钮背景变为深色高亮。
- 图标使用蓝色或绿色。
- 左侧有 2px 高亮条。
- 顶部或画布状态栏显示当前工具名。

禁用状态：

- 未导入图片时，除导入、打开项目、智能抠图上传入口外，区域/画笔类工具禁用。
- 禁用图标透明度降低，hover tooltip 显示原因：`请先导入图片`。

绘制中状态：

- 当前绘制工具保持高亮。
- 画布 cursor 切换为 crosshair。
- 画布状态栏显示当前绘制尺寸。

抠图处理中状态：

- 智能抠图按钮显示小型进度环。
- 打开模态后，主编辑器快捷键不抢占抠图模块。

### 5.3 当前实现调整建议

当前 `LeftSidebar.vue` 同时承担上传、智能抠图、工具选择、矩形预设尺寸、文字列表。为了避免重复和减轻左侧负担，建议第一轮这样迁移：

保留在左侧：

- 导入图标按钮。
- 选择、矩形、圆角矩形、多边形、魔棒、文字、画笔、橡皮、智能抠图。
- 可选更多菜单：圆形、三角形、菱形、星形、心形。

迁出左侧：

- 文字列表迁到“对象”Tab。
- 预设尺寸迁到“批量”Tab，作为“快捷区域尺寸”。导出 Tab 只放输出尺寸和平台尺寸预设。
- 上传成功的 `已就绪` 文案改为顶部或状态栏摘要，不在左侧占一行。

视觉调整：

- 左侧宽度从 200px 改为约 82px。
- 工具按钮显示图标 + 文字，tooltip 作为补充说明。
- 如果暂时没有图标库，可以先用统一字符图标，但不要使用 emoji。
- 智能抠图和导入图片放在左侧最上方两个按钮，使用普通工具按钮样式，避免视觉权重过高。

## 6. 中央画布工作区

### 6.1 功能定位

中央画布负责视觉确认和直接操作，不承担复杂配置表单。

包含：

- 画布。
- 棋盘格透明背景。
- 顶部/左侧标尺。
- 图层内容。
- 活动图层外框和控制点。
- 裁剪区域遮罩、边框和名称。
- 参考线。
- 文字框。
- 画布空状态。
- 画布状态栏。

### 6.2 空状态

无图片时：

- 中央显示低对比拖拽区域。
- 主文本：`拖入图片开始切图`。
- 次文本：`支持 PNG / JPG / WebP`。
- 主按钮：`导入图片`。
- 不展示工具说明长文案。
- 右侧面板显示空状态摘要，不展示大量禁用表单。

### 6.3 图片加载状态

导入处理中：

- 画布中央显示小型进度和文件名。
- 左侧工具暂时禁用。
- 右侧面板显示 `正在载入图片...`。

导入成功：

- 图片自动适配画布。
- 活动图层显示蓝色外框。
- 画布状态栏显示缩放比例和图片尺寸。

导入失败：

- 画布显示错误提示。
- 提供重新导入按钮。
- 错误不进入图层列表。

### 6.4 图层显示状态

普通图层：

- 按图层列表顺序显示，列表顶部为视觉最上层。
- 非活动图层不显示控制点，只在选中或多选时显示细边框。

活动图层：

- 显示蓝色虚线外框。
- 显示 8 个缩放控制点。
- 左上角显示图层名，使用低干扰标签。
- 活动图层不改变图片层级。

隐藏图层：

- 不在画布显示。
- 右侧对象列表中保留，眼睛图标变为关闭状态。

多选图层：

- 每个选中图层显示青色虚线边框。
- 活动图层仍显示更强外框。

### 6.5 裁剪区域状态

普通区域：

- 边框为白色半透明。
- 内部轻微透明覆盖，不遮挡图片细节。
- 标签显示 `名称 / 宽x高`，在区域上方或左上角。

选中区域：

- 边框为亮蓝色。
- 显示 8 个控制点。
- 标签背景变为蓝色，文字清晰。
- 右侧“属性”Tab 自动显示区域属性。

多选区域：

- 所有选中区域显示浅蓝边框。
- 主选中区域显示控制点。
- 画布状态栏显示 `已选 5 个区域`。
- 右侧“属性”Tab 显示批量属性摘要和批量变换入口。

区域越界：

- 边框或标签显示琥珀色提示。
- 导出检查中记录该区域。
- 不直接阻止编辑。

区域无内容：

- 边框显示红色警示标记。
- 导出检查中标为错误或高风险警告。

### 6.6 参考线状态

普通参考线：

- 青蓝色 1px 虚线。
- 鼠标 hover 时变亮。

新增参考线：

- Ctrl/Command + 点击标尺生成。
- 新增瞬间短暂闪烁，帮助用户确认。

选中或 hover 参考线：

- 显示位置数值标签，例如 `Y: 480`。
- 右键可删除。

无参考线：

- 右侧“批量”Tab 的参考线切片区域显示空状态：`从标尺添加参考线后可生成切片`。

### 6.7 画布状态栏

显示：

- 缩放比例。
- 鼠标坐标。
- 当前区域尺寸。

## 7. 右侧任务面板

### 7.1 面板结构

右侧任务面板使用“上方 Tabs + 下方清单面板”的上下结构，避免所有功能挤在一个长滚动里，同时让交付对象列表始终可见。

推荐 Tabs：

1. 属性。
2. 批量。
3. 模板。
4. 导出。

卡片列表不再作为顶部 Tab，而是固定在右侧底部：

- 卡片列表面板显示裁剪框卡片列表和文字框卡片列表。
- 裁剪框和文字框必须分组展示，不能混在同一列表中。
- 卡片列表顶部提供搜索输入框；裁剪框只搜索名称，文字框只搜索文本内容。
- 卡片列表上边缘可拖动，拖动范围为右侧总高度的 20%-80%。
- 卡片列表面板支持一键收起和展开。

默认显示规则：

- 未选择对象：默认打开“批量”或“导出”，取决于是否已有区域。
- 点击裁剪框、文字框或新生成裁剪框时，不自动切换 Tab。
- 用户手动停留在哪个 Tab，就保持在哪个 Tab。
- 需要编辑属性时，由用户手动切到“属性”Tab。

### 7.1.1 第一轮 Tab 精简建议

当前推荐第一轮直接做 4 个 Tabs：

1. 属性。
2. 批量。
3. 模板。
4. 导出。

其中：

- 图层列表先保留在“属性”Tab 顶部，分组标题使用“图层”。
- 区域卡片列表和文字框卡片列表固定在右侧底部可拖拽面板。
- 不再计划把“卡片列表”作为顶部 Tab，避免和生产配置 Tab 抢位置。

4 个 Tabs 过渡方案的约束：

- 图层显隐、重命名、排序、删除必须仍然一眼可找到，不能埋到滚动很深的位置。
- 文字对象列表迁出左侧后，必须能在底部卡片列表面板中选中和定位文字。
- 区域卡片列表必须保留勾选导出、删除、清空、选中联动画布的能力。
- 批量、模板、导出三个生产主流程不能被卡片列表内容挤到不可见区域。

### 7.1.2 Tab 默认打开策略

结合当前操作习惯，建议：

- 刚导入图片：打开“批量”Tab。
- 选中区域：保持当前 Tab，不自动跳转。
- 选中文字：保持当前 Tab，不自动跳转。
- 点击模板入口：打开“模板”Tab。
- 点击导出检查摘要：打开“导出”Tab 并滚到检查区域。
- 用户手动切换 Tab 后，后续选择对象也不自动跳 Tab。

### 7.2 图层分组

功能定位：

- 管理图层。
- 不管理裁剪区域清单，裁剪区域归入“清单”Tab。

包含：

- 图层列表。
- 图层显示/隐藏。
- 图层重命名。
- 图层排序。
- 图层删除。

图层行状态：

- 普通：显示缩略图、图层名、尺寸摘要、可见性图标。
- 活动：整行高亮，左侧有蓝色标记。
- 隐藏：缩略图变暗，名称透明度降低。
- 拖拽排序：目标位置显示插入线。
- 重命名：名称位置变为输入框，Enter 保存，Esc 取消。
- 删除确认：点击删除后显示二次确认或轻量确认弹层。

空状态：

- 无图层时显示 `导入图片后会显示图层`。
- 提供一个 `导入图片` 快捷按钮，但不显示完整导入说明。

开发提示：

- 当前图层管理代码在 `RightSidebar.vue`，可先抽成 `ObjectPanel.vue`。
- 当前文字列表在 `LeftSidebar.vue`，迁移后通过 `editor.textAnnotations` 读取。
- 不要改 `editor.layers` 的排序语义，列表顶部仍代表视觉最上层。

### 7.3 属性 Tab

功能定位：

- 编辑当前选中对象。
- 根据选中对象类型切换内容。
- 不显示批量生成、模板和完整导出配置。

无选中对象：

- 显示摘要：当前工具、区域数量、活动图层。
- 提示 `选择区域、文字或图层以编辑属性`。
- 不显示禁用输入框堆叠。

选中单个区域：

- 显示区域名。
- 显示形状只读或可轻量切换。
- 显示 X/Y/宽/高。
- 圆角矩形显示圆角输入。
- 显示单区域预览按钮。
- 显示单区域导出按钮，但按钮文案为 `导出此区域`，并使用导出 Tab 的命名规则摘要。
- 不显示完整导出尺寸、格式、命名表单。

选中多个区域：

- 显示 `已选 N 个区域`。
- 显示外接矩形尺寸。
- 提供批量移动、删除、翻转、旋转。
- 不显示单个区域名称输入。
- 显示 `批量预览所选` 快捷按钮，但完整导出仍在底部交付栏。

选中文字：

- 显示文字内容。
- 显示字号、字重、颜色。
- 显示文字框 X/Y/宽/高。
- 不显示区域形状和导出按钮。

选中图层：

- 显示图层名。
- 显示位置 X/Y。
- 显示缩放比例。
- 显示可见性。
- 显示是否使用原图/编辑后工作画布摘要。
- 不重复图层列表。

工具参数状态：

- 当当前工具为画笔时，属性 Tab 顶部显示画笔大小和颜色。
- 当前工具为橡皮时，显示橡皮大小。
- 当前工具为魔棒时，显示容差。
- 如果已选对象，工具参数折叠到“当前工具设置”小节，避免盖过对象属性。

开发提示：

- 当前属性、文字属性、工具参数和区域变换都在 `RightSidebar.vue`。
- 第一轮可以先不改变数据逻辑，只把这些 section 分组到 `PropertiesPanel.vue`。
- 单区域导出按钮可以保留，但视觉上降级为次按钮；主导出仍在底部交付栏。
- 多选区域时，不要显示单区域名称输入，避免误导用户。

### 7.4 批量 Tab

功能定位：

- 生成裁剪区域。
- 管理参考线切片。
- 不保存模板，不设置导出格式。

分组：

- 网格生成。
- 等距复制。
- 长图切片。
- 参考线切片。

网格生成状态：

- 默认折叠或半展开，显示行列和单元尺寸摘要。
- 输入：行数、列数、起始 X/Y、单元宽高、间距、命名前缀、圆角。
- 操作按钮：`生成 N 个区域`。
- 替换现有区域作为危险选项，放在按钮上方，默认关闭。
- 如果生成数量超过阈值，例如 100，按钮旁显示性能提示。

等距复制状态：

- 未选中区域：禁用并显示 `选择一个区域后可复制`。
- 选中单区域：可配置方向、数量、间距或自定义偏移。
- 选中多区域：默认禁用，提示 `等距复制仅支持主选中区域`，避免歧义。

长图切片状态：

- 无活动图层：禁用，提示 `请先选择活动图层`。
- 有活动图层：显示切片高度、开始 Y、结束 Y、重叠、命名前缀。
- 输入非法：显示行内错误，例如 `重叠值必须小于切片高度`。
- 生成后：显示 `已生成 N 个切片` 的短暂成功状态。

参考线切片状态：

- 无参考线：显示空状态和操作提示。
- 有参考线：显示 `横向 X 条 / 纵向 Y 条`。
- 提供 `从参考线生成区域`。
- 提供 `清空参考线`，但使用危险样式。

开发提示：

- 当前 `BatchCutPanel.vue` 可以直接放入批量 Tab。
- 先统一 details/summary 的视觉，不必重写生成逻辑。
- “预设尺寸”如果从左侧迁入，建议作为批量 Tab 的第一组“快捷区域尺寸”，点击后沿用现有 `createPresetRegion` 的生成语义。
- 快捷区域尺寸不应与导出尺寸混淆：文案使用“创建裁剪框尺寸”，不要叫“输出尺寸”。

### 7.5 模板 Tab

功能定位：

顶部状态：

保存模板状态：

- 如果没有区域：保存按钮禁用，提示 `至少需要 1 个区域`。
- 如果没有活动图层：保存按钮禁用，提示 `请先选择活动图层`。
- 有区域时：显示模板名输入。
- 可选项：保存导出配置，默认关闭或按 Pro 规划开启。
- 点击保存后，模板出现在列表顶部。

模板项状态：

- 普通：显示模板名、区域数、尺寸摘要、最近使用时间。
- hover：显示套用、重命名、删除图标。
- 选中：整行高亮，显示详情摘要。
- 套用中：按钮显示 loading。
- 套用成功：短暂显示绿色成功状态。

模板详情：

- 区域数量。

- 是否包含导出配置。
- 命名规则摘要。
- 最后使用时间。

套用模板：

- 主按钮：`套用到当前图层`。

- 如果做搜索框但没有搜索逻辑，会降低信任；宁愿先不展示。
- “保存导出配置”如果已经存在，就保留在模板保存区；不要在导出 Tab 重复一个“保存为模板”完整表单。

输出尺寸状态：

- 统一尺寸开关。
- 开启统一尺寸后显示宽高、适配方式、背景色。
- `contain` 模式显示背景色。

文件命名状态：

- 命名规则输入。

- 前 3 项预览。
- 未知变量显示行内警告。

增强状态：

- 导出体检的开关项和帮助说明必须保留在导出 Tab，顶部栏和底部栏只显示体检摘要与跳转，不替代详情配置。

## 8. 底部固定交付栏

### 8.1 功能定位

底部交付栏是最终动作区，始终可见。

包含：

- 选中/勾选数量。
- 导出检查状态。
- 批量预览按钮。
- 导出 ZIP 主按钮。

不包含：

- 完整导出设置。
- 模板列表。
- 批量生成参数。

### 8.2 状态描述

无图片：

- 显示 `导入图片后可生成区域并导出`。
- 批量预览和导出按钮禁用。

有图片无区域：

- 显示 `暂无裁剪区域`。
- 批量预览和导出按钮禁用。
- 提供 `去批量生成` 文本按钮。

有区域未勾选：

- 显示 `将导出全部 N 项`。
- 批量预览可用。
- 导出按钮根据检查结果决定是否可用。

有区域且已勾选：

- 显示 `将导出已勾选 N / 共 M 项`。

- 导出只导出勾选区域。

导出检查警告：

- 显示琥珀色 `N 个警告`。
- 导出按钮可用，但点击前弹出轻量确认或显示检查详情。

导出检查错误：

- 显示红色 `无法导出`。

导出中：

- 导出按钮显示进度和 loading。

导出完成：

- 显示绿色 `已导出 ZIP`。

- 执行批量导出。
- 打开导出检查详情。
- 跳到右侧导出 Tab。

实现建议：

- 迁移底部栏前，必须先去掉 `PreviewModal.vue` 中通过 `document.querySelector('.export-btn')` 触发导出的方式；否则右侧旧按钮移除后，预览弹窗内的导出会失效。
- 第一轮可以把 `handleBatchExport` 仍保留在 `RightSidebar.vue`，但通过事件传给 `DeliveryBar` 和 `PreviewModal`。
- 更好的做法是先抽成 `useBatchExportAction.ts`，让 `RightSidebar`、`DeliveryBar`、`PreviewModal` 调同一个动作，再迁移按钮位置。
- 只在统一导出动作完成后，才移除右侧滚动内容里的批量导出主按钮。

## 9. 批量预览弹窗

### 9.1 功能定位

批量预览是导出前确认，不是另一个编辑面板。

包含：

- 缩略图网格。
- 文件名预览。
- 尺寸。
- 警告标记。
- 放大查看。
- 确认导出。

不包含：

- 完整导出设置。
- 模板管理。
- 批量生成表单。

### 9.2 状态描述

打开加载：

- 显示 `正在生成预览...`。
- 按区域数量显示进度。

正常：

- 网格显示缩略图。
- 每张卡片显示导出文件名和尺寸。
- 点击缩略图进入放大预览。

有警告：

- 卡片右上角显示警告图标。
- hover 或点击显示原因。
- 顶部显示 `有 N 个警告`。

无可预览：

- 如果没有区域，弹窗不应打开。
- 底部栏按钮应提前禁用。

导出中：

- 弹窗按钮显示 loading。
- 不允许关闭或关闭前提示导出中。

开发提示：

- 批量预览弹窗的确认导出必须通过显式事件或回调触发统一导出动作，不允许再查询 `.export-btn`。
- 批量预览应使用和底部交付栏一致的目标区域集合：有勾选时只预览勾选区域，未勾选时预览全部区域。
- 文件名预览、尺寸和警告应来自导出 Tab 的当前配置，不在弹窗内重复配置表单。

## 10. 智能抠图工作区

### 10.1 功能定位

智能抠图使用独立模态工作区，避免主界面同时出现两套编辑逻辑。

入口：

- 左侧工具栏智能抠图按钮。
- 当前图层上下文中可显示 `使用当前图层抠图` 快捷入口，但不展开完整抠图配置。

### 10.2 状态描述

未选择来源：

- 显示上传图片、使用当前图层原图、使用当前图层编辑后三个入口。

模型加载：

- 显示模型类型、加载进度、本地缓存提示。

推理中：

- 显示进度、取消按钮。
- 主画布快捷键暂停响应。

遮罩编辑：

- 左侧为保留/去除画笔。
- 中央为抠图结果预览。
- 右侧为边缘精修。
- 不显示主编辑器的裁剪区域和导出设置。

完成：

- 主按钮：发送到画布。
- 次按钮：导出透明 PNG。
- 发送到画布后，结果成为新图层，回到主工作区。

失败：

- 显示错误原因。
- 提供重新推理或换模型。

## 14. 开发约束

### 14.1 业务逻辑约束

- 不改变图层数组顺序语义：列表顶部仍是视觉最上层。
- 不改变区域选择、多选、勾选导出的语义。
- 不改变模板坐标换算逻辑。
- 不改变导出合成逻辑。
- 不改变智能抠图工作区的数据流。
- 不新增多图队列。

### 14.2 状态管理约束

- 优先复用现有 Pinia store：`editor`、`export`、`history`、`matting`。
- 右侧 Tab 当前激活状态可以先放组件本地状态。
- 如果多个组件需要打开同一个预览或导出动作，优先抽 composable，不要 DOM 查询。
- 不为纯视觉状态新增复杂全局 store。

### 14.3 样式约束

- 新增全局 UI token，减少每个组件 scoped style 重复定义。
- 卡片圆角不超过 6px，弹窗不超过 8px。
- 工具按钮、图标按钮必须有固定宽高，避免文字和图标导致布局跳动。
- 列表行高度固定，区域名过长使用省略号。
- 所有主按钮同一颜色，同一高度。
- 危险操作只用红色描边或红色文字，不用大面积红色块。

### 14.4 可访问性和效率约束

- 图标按钮必须有 `title` 或 tooltip。
- 禁用按钮必须能解释原因。
- 导出中禁止重复点击。
- 删除、清空、替换区域需要确认或可撤销。
- 高频动作不要隐藏到二级菜单：批量预览、导出、套用模板、生成区域必须一眼能找到。
